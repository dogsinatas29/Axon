# AXON: 자동화 소프트웨어 공장 (Phase 07)
[English Version](README.md)

![AXON Concept](https://raw.githubusercontent.com/dogsinatas29/Axon/master/axon%EA%B0%9C%EB%85%90.png)

AXON은 최소한의 개입으로 요구 사양서를 실제 동작하는 코드로 변환하기 위해 설계된 고성능 멀티 에이전트 AI 코딩 공장입니다. Phase 07은 **로컬라이징(현지화)**, **강력한 격리(Isolation)**, 그리고 **추론 최적화**에 집중했습니다.

## 🚀 핵심 기능 및 업데이트

### 1. 전면 터미널 로컬라이징
- **다국어 지원** 도입 (한국어, 영어, 일본어).
- 모든 CLI 프롬프트, 요원 모집 메시지, 시스템 로그가 사용자 설정에 따라 동적으로 번역되어 출력됩니다.
- 언어 설정은 `axon_config.json`에 영구 저장됩니다.

### 2. 지능형 코드 추출 엔진
- **언어 중립적 추출**: 에이전트의 제안서에서 언어(Python, JS, Rust 등)에 상관없이 코드 블록을 자동으로 감지하고 추출합니다.
- **파일명 자동 감지**: 코드 내 주석(예: `# main.py` 또는 `// server.js`)을 스캔하여 원본 파일 구조를 복원합니다.
- **격리 샌드박스**: 프로젝트별 독립 디렉토리(예: `./GEMINI/src/`)를 자동 생성하여 작업 공간을 깔끔하게 관리합니다.

### 3. 아키텍트 CTO 지능 강화
- **전략적 분해**: 아키텍트의 페르소나를 **CTO 급**으로 격상하여, 거대한 사양서를 원자 단위의 병렬 가능한 태스크로 정교하게 분해하도록 개선했습니다.
- **추론 최적화**: 로컬 LLM(Ollama/Mistral) 환경에서의 지연 시간을 줄이기 위해 시스템 프롬프트를 경량화하여 속도를 최대 2배 개선했습니다.

### 4. 안정성 및 UI 동기화
- **OS 에러 36 해결**: 부트스트랩 과정에서 발생하던 '파일명 너무 긺' 오류를 수정했습니다.
- **실시간 상태 동기화**: 쓰레드 상태 전파 로직을 교정하여 스튜디오 UI에서 '완료' 상태 및 대시보드 수치가 즉각 반영되도록 했습니다.

## 🏛️ Sovereign Agent Protocol: 요원별 역할 및 사고 체계

AXON은 생산 등급의 결과물을 보장하기 위해 각 요원에게 엄격한 위계와 특화된 사고 프레임워크를 강제합니다.

### 👑 1. 아키텍트 (Architect / CTO)
- **역할**: 전략적 기획 및 시스템 전체 설계.
- **사고 방식**: **전략적 분해 (Stage-based COT)**. 단일 진실 공급원(SSOT)과 모듈형 확장성(Hub->Cluster->Node)을 중심으로 사고하도록 설계되었습니다.
- **책임**: 마스터 아키텍처를 생성하고, 이를 원자 단위의 병렬 가능한 태스크로 분해합니다.

### 👴 2. 시니어 (Senior / Tech Lead)
- **역할**: 품질 보증 및 엄격한 코드 리뷰.
- **사고 방식**: **비판적 분석 (Adversarial Analysis)**. '의심 우선' 모드로 동작하여 환각 증세나 누락된 로직을 찾아냅니다.
- **책임**: 주니어의 제안을 승인하거나 반려합니다. "코드 블록 없음 = 즉시 반려" 규칙을 집행합니다.

### 👶 3. 주니어 (Junior / Developer)
- **역할**: 순수 구현 및 코딩.
- **사고 방식**: **직렬적 실행 (No-Preamble)**. 불필요한 사고 과정을 배제하고 아키텍트의 가이드에 따라 코드 생산에만 100% 집중합니다.
- **책임**: 할당된 태스크에 대해 전체 소스 코드와 기술적 변경 사항(Diff)을 제출합니다.

## 🛠️ 시작하기

```bash
# 공장 빌드
cargo build --release

# 사양서와 함께 즉시 실행
./target/release/axon-daemon run GEMINI.md

# 대화형 실행
./target/release/axon-daemon run
```

## 📋 릴리즈 노트: v0.0.17 - 제어 및 격리 (Control & Isolation)

### 🚀 주요 기능
- **멀티 에이전트 오케스트레이션**: `주니어 -> 시니어 -> 아키텍트` 명령 체계를 정립하고 라운드 로빈 스케줄링을 지원합니다.
- **Ollama 런타임 어댑터**: 로컬 모델 구동을 위한 기본 지원 및 성능 추적 기능을 통합했습니다.
- **관측성 및 리포팅**: 실행 경로에 대한 실시간 메트릭 수집 및 이벤트 버스 통합을 완료했습니다.
- **견고한 부트스트랩 프로토콜**: 설정 및 컨텍스트 구축을 위한 단계별 초기화 프로세스를 구축했습니다.

### 🛠️ 기술적 변경 사항
- **Core**: `ObservabilityReport` 및 `RuntimeMetrics`를 저장소 및 에이전트 로직에 추가했습니다.
- **Model Driver**: 구조화된 메트릭을 반환하도록 트레이트(Trait)를 업데이트했습니다.
- **Daemon**: 계층 기반 폴백(Fallback) 및 태스크-샌드박스 동기화 로직을 구현했습니다.
