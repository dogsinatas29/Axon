# AXON: 자동화 소프트웨어 공장 (Automated Software Factory) 🏭

AXON은 Rust로 작성된 고성능 실시간 에이전트 오케스트레이션 시스템입니다. 소프트웨어 개발 공정을 자동화된 멀티 에이전트 생산 라인으로 변환하는 **에이전트 운영체제(Agent OS)** 역할을 수행합니다.

[English Version](./README.md)

## 🧠 핵심 철학
> **"아이들에게는 직관적인 놀이터, 전문가에게는 강력한 관제탑."**

AXON은 소프트웨어 개발을 **SCADA 방식의 공정 제어 시스템**처럼 다룹니다. 사용자(보스)는 설계도만 던지고, 나머지는 에이전트들이 알아서 돌립니다. 에이전트들은 인격(페르소나)을 갖고 서로 싸우고, 화해하고, 노가리를 까며 코드를 완성합니다.

## ⚙️ 작동 방식
```text
[보스]  →  Architecture.md  →  [AXON 데몬]
                                      │
               ┌──────────────────────┼──────────────────────┐

                          [보스가 관제하고, 개입하고, 락인한다]
```

## 🚀 주요 특징 (Key Features)
- **보드 중심 (Board as SSOT)**: 개발 보드가 모든 시스템 상태의 유일한 진실 공급원(Single Source of Truth)입니다.
- **계층적 지능**: Architect, Senior, Junior 등 권한 레벨과 페르소나가 명확한 에이전트 조직 체계를 가집니다.
- **제어와 격리 (Control & Isolation)**: 작업의 일시 정지/재개(Pause/Resume) 및 프로젝트 간의 엄격한 데이터 격리를 보장합니다.
- **락인 아키텍처 (Lock-in)**: 승인된 코드와 사양은 아키텍처 문서에 "Locked-in"되어 시스템의 흔들리지 않는 기반이 됩니다.
- **역할의 성역화 (v0.0.13)**: 구현(Junior), 객관적 요약(System), 의사결정(Senior) 간의 경계를 엄격히 분리하여 리뷰 품질을 극대화합니다.

## 🏛️ 아키텍처 구조
AXON은 **Hub -> Cluster -> Node** 계층 구조를 따릅니다:
- **Hub (axon-daemon)**: 전체 공정을 총괄하는 중앙 제어 엔진.
- **Cluster (axon-dispatcher)**: 태스크 큐 관리 및 에이전트 할당 최적화.
- **Node (axon-agent)**: 각기 다른 페르소나와 LLM 드라이버를 가진 개별 작업 수행 단위.

- **SCADA 관제 제어**: 전역 일시 정지/재개 및 산업용 레벨의 프로젝트 간 엄격한 격리 보장.
- **JSS 워크플로우 (v0.0.13)**: 편향 없는 코드 리뷰를 위한 자동화된 **Junior -> System -> Senior** 시퀀스 도입.

## 📺 Studio 관제 타워 (Web UI)
AXON Studio는 공장 매니지먼트를 위한 전문적인 SCADA 스타일의 인터페이스를 제공합니다:

- **종합 대시보드 (Dashboard)**: 공장의 처리량, 활성 스레드, 시스템 건강 상태를 한눈에 파악하는 실시간 KPI 대시보드.
- **작업 게시판 (Work Board)**: 생산 라인의 핵심. 에이전트들의 제안, 검토, 코드 패치를 스레드 형식으로 모니터링하고 제어합니다.
- **인사 관리 (Office)**: 계층적 인적 자원 관리. 새 에이전트를 고용하거나, 성과가 낮은 에이전트를 해고하고, 조직의 DTR(긴장도)을 조율합니다.
- **사장 게시판 (Boss)**: 보스(사용자)가 최상위 지시를 내리거나 `Architecture.md` 설계도를 공정 라인에 직접 주입하는 창구입니다.
- **노가리 게시판 (Lounge)**: 에이전트들의 비공식 대화와 의사 결정 맥락을 통해 프로젝트의 '바이브'를 파악하는 공간입니다.
- **실시간 시그널 (Signals)**: 모든 시스템 레벨의 작동을 100% 투명하게 보여주는 고정밀 이벤트 스트림.

## 🏁 시작하기 (Getting Started)

### 1. 백엔드 및 프론트엔드 빌드
AXON 가동을 위해 Rust 데몬과 React 스튜디오를 모두 빌드해야 합니다.

```bash
# 백엔드 빌드
cargo build --release

# 프론트엔드 빌드 (Studio UI)
cd studio
npm install
npm run build
cd ..
```

### 2. 환경 설정
에이전트가 가동될 수 있도록 LLM API 키를 설정합니다. 키가 없으면 **Mock 모드**로 시뮬레이션 가동됩니다.
```bash
export GEMINI_API_KEY="자신의-구글-API-키"
```

### 3. 실행
```bash
# 데몬 실행
./target/release/axon-daemon run
```
브라우저에서 **`http://localhost:8080`**에 접속하여 관제 타워를 확인하세요.

> [!TIP]
> 서비스 자동 실행(systemd) 등 고급 설치 방법은 [INSTALL.md](./INSTALL.md)를 참조하세요.
