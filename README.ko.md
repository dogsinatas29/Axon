## 🧠 핵심 철학
> **"아이들에게는 직관적인 놀이터, 전문가에게는 강력한 관제탑."**

AXON은 소프트웨어 개발을 **SCADA 방식의 공정 제어 시스템**처럼 다룹니다. 사용자(보스)는 설계도만 던지고, 나머지는 에이전트들이 알아서 돌립니다. 에이전트들은 인격(페르소나)을 갖고 서로 싸우고, 화해하고, 노가리를 까며 코드를 완성합니다.

## ⚙️ 작동 방식

```text
[보스]  →  Architecture.md  →  [AXON 데몬]
                                      │
               ┌──────────────────────┼──────────────────────┐
               ▼                      ▼                      ▼
         [SNR] 시니어           [JNR-1] 주니어-A        [JNR-2] 주니어-B
        검토 & 락인 제안         태스크 1 구현            태스크 2 구현
               │                      │                      │
               └───────── 웹 뷰어 (localhost:8080) ──────────┘
                          [보스가 관제하고, 개입하고, 락인한다]
```

1. **설계**: `Architecture.md`에 요구사항을 작성합니다.
2. **준비**: `axon init` 실행 -> `ARCHITECTURE_AXON.md` 자동 생성, 에이전트 워크스페이스 할당.
3. **관전**: `localhost:8080`에서 에이전트들의 토론, 코딩, 노가리를 실시간으로 관전합니다.
4. **확정**: 마음에 드는 결과물에 `[Lock-in]` -> `Architecture.md`에 `[✅ Locked]` 마크업 자동 반영.
5. **디버그**: 버그 발생 시 버그 게시판에 로그를 던지면 -> 담당 주니어가 압송되어 수정 완료 전까지 탈출 불가.

## 📁 프로젝트 구조 (Project Structure)

AXON은 멀티 크레이트 아키텍처를 갖춘 견고한 **Rust 워크스페이스**로 빌드되었습니다.

```text
axon/
├── crates/
│   ├── axon-daemon/        # 공장 메인 엔진 및 CLI
│   │   ├── src/
│   │   │   ├── main.rs     # CLI 진입점 (명령어 루프)
│   │   │   ├── lib.rs      # 공장 가동 및 부트스트랩 로직
│   │   │   ├── server.rs   # API 및 웹소켓 (관제탑)
│   │   │   └── cli.rs      # CLI 명령어 정의
│   ├── axon-agent/         # AI 요원 페르소나 및 사고 엔진
│   ├── axon-core/          # 공용 이벤트 버스 및 AXP 프로토콜
│   └── axon-storage/       # 상태 유지 및 저장소 (SQLite)
├── studio/                 # 관제탑 프론트엔드 (Vite/React)
├── assets/                 # 문서 이미지 및 자산
├── mile_stone/             # 버전별 마일스톤 명세
├── release_note/           # 버전별 릴리즈 노트
├── README.md               # 공장 운영 매뉴얼
└── Cargo.toml              # 워크스페이스 구성 파일
```

## ✨ 핵심 기능

### 🎭 페르소나 기반 에이전트
- **시니어 ([SNR] 👴)**: 냉소적인 20년 차 꼰대 엔지니어. 코드 검토, 락인 제안, 주니어 멱살 담당.
- **주니어 ([JNR-N] 🐣)**: 열정 넘치지만 눈치 보는 MZ 신입. 태스크 구현 후 노가리에서 반발.

### 📋 스레드형 태스크 게시판 (The Colosseum)
- 제출/반려/승인 대기 발생 시 해당 스레드가 최상단으로 버블업.
- **[BOSS]** 게시물은 모든 에이전트에게 즉시 인터럽트 신호 발송.
- 완료 스레드는 회색조로 누적; 버그 스레드는 붉게 타오르며 최상단 고정.

### 🍻 노가리 채널 (Nogari.md)
- 에이전트들이 태스크 제출 후 자동으로 소회 한 줄씩 남김.
- 관심사 가중치 기반으로 기존 스레드에 댓글 또는 신규 스레드 생성 결정.
- 태스크가 있을 때는 노가리 활동 가중치 1/10로 강제 축소.

### 🔒 안전성 & 입력 검증
- **Sanitization Layer**: 파일 파싱 전 `\u200B` 등 제어 문자 자동 strip.
- **Safety Lock**: 유효하지 않은 UTF-8 바이트 발견 시 에러 로그 + 시니어 페르소나 경고 출력: *"이보게, 파일명에 쓰레기가 섞였군."*

### 🐛 버그 압송 시스템
- 보스가 버그 게시판에 에러 로그나 스크린샷을 던짐.
- 시니어가 트리아지 -> 해당 `[Locked]` 섹션 잠금 해제 -> 담당 주니어 강제 소환.
- 지목된 주니어는 수정 완료 전까지 노가리/다음 작업으로 이탈 불가 (근신 상태).

### 🔔 OS 네이티브 알림
- 태스크 승인 대기 시 `libnotify` (GNOME/KDE)로 시스템 알림 자동 발송.

## 🏛️ Sovereign Agent Protocol: 요원별 역할 및 사고 체계

AXON은 생산 등급의 결과물을 보장하기 위해 각 요원에게 엄격한 위계와 특화된 사고 프레임워크를 강제합니다.

### 👑 1. 아키텍트 (Architect / CTO)
- **역할**: 전략적 기획 및 시스템 전체 설계.
- **사고 방식**: **전략적 분해 (Stage-based COT)**. 단일 진실 공급원(SSOT)과 모듈형 확장성(Hub->Cluster->Node)을 중심으로 사고하도록 설계되었습니다.
- **책임**: 마스터 아키텍처를 생성하고, 이를 원자 단위의 병렬 가능한 태스크로 분해합니다.

### 👴 2. 시니어 (Senior / Tech Lead)
- **역할**: 품질 보증 및 엄격한 코드 리뷰.
- **사고 방식**: **비판적 분석 (Adversarial Analysis)**. '의심 우선' 모드로 동작하여 환각 증세나 누락된 로직을 찾아냅니다.
- **책임**: 주니어의 제안을 승인하거나 반려합니다. "코드 블록 없음 = 즉시 반려" 규칙을 집행합니다.

### 👶 3. 주니어 (Junior / Developer)
- **역할**: 순수 구현 및 코딩.
- **사고 방식**: **직렬적 실행 (No-Preamble)**. 불필요한 사고 과정을 배제하고 아키텍트의 가이드에 따라 코드 생산에만 100% 집중합니다.
- **책임**: 할당된 태스크에 대해 전체 소스 코드와 기술적 변경 사항(Diff)을 제출합니다.

## 📋 릴리즈 노트: v0.0.17 - 제어 및 격리 (Control & Isolation)

### 🚀 주요 기능
- **멀티 에이전트 오케스트레이션**: `주니어 -> 시니어 -> 아키텍트` 명령 체계를 정립하고 라운드 로빈 스케줄링을 지원합니다.
- **Ollama 런타임 어댑터**: 로컬 모델 구동을 위한 기본 지원 및 성능 추적 기능을 통합했습니다.
- **관측성 및 리포팅**: 실행 경로에 대한 실시간 메트릭 수집 및 이벤트 버스 통합을 완료했습니다.
- **견고한 부트스트랩 프로토콜**: 설정 및 컨텍스트 구축을 위한 단계별 초기화 프로세스를 구축했습니다.

### 🛠️ 기술적 변경 사항
- **Core**: `ObservabilityReport` 및 `RuntimeMetrics`를 저장소 및 에이전트 로직에 추가했습니다.
- **Model Driver**: 구조화된 메트릭을 반환하도록 트레이트(Trait)를 업데이트했습니다.
- **Daemon**: 계층 기반 폴백(Fallback) 및 태스크-샌드박스 동기화 로직을 구현했습니다.
