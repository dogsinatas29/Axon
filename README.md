# AXON: Agent Orchestration Framework

AXON is an autonomous software production system where AI agents (Architects, Seniors, and Juniors) collaborate to transform specifications into high-quality code. 

## 🚀 Principles

1.  **Architecture-First**: No code is written without a validated `architecture.md` (SSOT), obeying the Hub -> Cluster -> Node Sovereign Protocol.
2.  **Role Sanctity**: Strict hierarchical boundaries ensure quality and accountability.
3.  **Stateless Execution**: Agents operate as interchangeable factory workers.
4.  **Reasoning Protocols (COT/TOT)**: Agents must explicitly generate `<thought>` and `<analysis>` blocks before outputting any code or architecture.

## 🏭 Quick Start & Usage Guide

AXON transforms a simple markdown requirement file into a fully working project while visually showing the team's live activity via the Studio GUI.

### 1. Preparation
Set your API keys as environment variables:
```bash
export GEMINI_API_KEY=your_key
# Optional: export CLAUDE_API_KEY=your_key
```

### 2. Start the Daemon
Compile and push the system into operation mode:
```bash
cargo build --release
../target/release/axon-daemon run
```

### 3. Agent Recruitment (The Office)
The terminal will present an interactive menu to assign AI models to specific Factory Roles:
- **Architect (CTO)**: 1 core AI to read the user's base spec and design the `Architecture.md` (Master Hub).
- **Seniors**: Reviewers who ruthlessly critique code.
- **Juniors**: Grunts who write code for terminal task threads.

*Select the brand (e.g., Gemini) and the model version using the interactive number prompts.*

### 4. Spec Ingestion
Provide the exact path to your specification file when prompted:
```
Enter Specification File Path (e.g., GEMINI.md): TEST/GEMINI.md
```

### 5. Monitor via Studio UI
Once the daemon runs and the Architect begins analyzing, immediately open your browser:
**URL:** [http://localhost:8080](http://localhost:8080)

1. **Dashboard**: View overall signal counts and metrics.
2. **Work Board**: Within ~30 seconds of launching, the Architect will finish parsing the Spec and **automatically populate the Work Board** with individual Task Threads. Click on a thread to see the parsed requirements.
3. **Signals**: Watch real-time logs as Juniors propose code and Seniors review it.

## 📝 Recent Milestones (v0.0.14)

- **Greedy Parsing & Domain Dominance Fix**: Refactored the daemon string parser to precisely extract code blocks without JSON spillovers. Added strict Prompt architecture enforcing the Hub -> Cluster -> Node rules over any given specific markdown specs.
- **COT/TOT Engine**: Dropped the 'DO NOT EXPLAIN' instructions, forcing Junior/Senior/Architect to explicitly print `<thought>` and `<reasoning>` tags before finalizing outputs.
- **UI Work Board Sync**: Overrode the `vec![]` hardcode; tasks generated by the Architect are now dynamically mirrored into `Thread` objects to instantly render on the web UI Work Board.
