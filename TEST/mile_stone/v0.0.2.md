# SYNAPSE Experimental Engine - v0.0.2

## Core Philosophy
Stable Core + Disposable Experiments

- Core (ECS, Renderer): Immutable, protected
- Experiments (Ghosts, Analysis): Disposable, isolated

---

## Architecture Model

[ Core ] <- (Event / Query) <- [ Ghosts / Experiments ]

- Core does not know external systems
- External systems cannot directly mutate Core

---

## Rules

### Rule 1 — Core Write Protection
External systems MUST NOT directly mutate Core state.

Allowed:
    emit(Event.Move(entity_id, dx, dy))

Forbidden:
    entity.position.x += 10

---

### Rule 2 — Ghost = Plugin

external_ghosts/
  ├── pathfinder/
  ├── ai_agent/
  └── debug_tools/

Constraints:
- Removable without side effects
- Minimal dependency on Core

---

### Rule 3 — Analysis Isolation

temp_analysis → tools/analysis/

Constraints:
- No runtime imports
- No execution path linkage

---

### Rule 4 — Project Root = Orchestrator Only

Allowed:
- init()
- wire()
- run_loop()

Forbidden:
- game logic
- AI logic
- physics logic

---

## Event System (Mandatory Layer)

class Event:
    pass

class Move(Event):
    def __init__(self, entity_id, dx, dy):
        pass

---

## Ghost Interface

class Ghost:
    def update(self, world, emit):
        pass

- world: read-only
- emit: only write path

---

## Design Constraints

1. Core must remain deterministic
2. All side effects go through Events
3. Experiments must be deletable at any time
4. No bidirectional dependencies

---

## Risk Control

Without these rules:
- State mutation becomes untraceable
- Debugging cost grows exponentially
- LLM/Agent integration destabilizes system

---

## Current Status (2026-03-23)

### Milestone: Reactive AI Agent ✅
The 1st stage of the "3-Stage AI Layer" has been successfully integrated.

1. **Stateless Reactive Logic**: Implemented `ReactiveGhost` in `external_ghosts/reactive_ghost/ghost.py`. It reacts to immediate stimuli (wall detection, player proximity) without internal state, maximizing determinism.
2. **Deterministic Response**: The AI uses `world_proxy.time` and `world_proxy.ticks` to drive its behaviors, ensuring that replaying the same scene produces identical AI intent.
3. **Execution Isolation**: Updated `World.update` to gate ghost execution, ensuring AI doesn't interfere with the injected input stream during playback/seek operations.
4. **Verified Toolchain**: The full pipeline (Record -> Seek -> Reactive AI React -> Replay -> Hash Check) is now operational and stable.
