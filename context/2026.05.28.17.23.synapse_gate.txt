1. 추론 과정 요약 (Core CoT)
- Ollama 모델(qwen2.5:7b 및 deepseek-coder-v2:lite)에서 VRAM 보호를 위해 OLLAMA_MUTEX를 사용하여 API 호출을 전역적으로 직렬화(Admission Control)하고 있음.
- 이때 각 에이전트(Agent)의 타임아웃 기본값이 300초(5분)로 설정되어 있어, 에이전트 간 병렬 요청이 겹치거나 단일 추론이 길어질 경우 대기 중인 다른 에이전트가 뮤텍스를 획득하기 전에 타임아웃되는 현상이 발생함.
- 이에 따라 에이전트 타임아웃 설정을 900초(15분)로 상향 조정하여 직렬화 병목으로 인한 타임아웃 오류를 방지함.
- 또한, TEST4_GTK 프로젝트에서 발생했던 이전 세션의 legacy state(잘못된 CWD 실행에 따른 db 불일치)를 정리하기 위해 TEST4_GTK/runtime/state.db를 최신화(checkpoint)한 후 루트 경로 runtime/state.db로 이관하여 SSOT(Single Source of Truth) 환경을 복구함.
- 추가적으로, 데몬이 루트 경로에서 구동될 때 TEST4_GTK 하위의 sandbox_root(TEST4_GTK/spec)를 정확히 바라보도록 sandbox_root 및 resume pipeline target_dir 결정 방식을 spec_path의 부모 디렉토리를 기준으로 참조하도록 수정함.
- 파일 타입 판별 로직(File Type Detection)에서 Lua 파일(.lua)을 단순한 C/C++ 소스 파일로 취급하여 시니어가 #include 누락 등의 환각(C++ 전용 검증 규칙 오적용)을 일으키고, 주니어가 잘못된 피드백에 의해 정상적인 코드를 제출하지 못하고 루프에 빠지는 현상을 해결하기 위해, 주니어의 규칙 블록(c_rule_block) 및 시니어의 검토 컨텍스트(review_context)에 `.lua` 파일에 대한 독립적인 전용 규칙 집합을 주입함.
- 시니어의 CoT (Chain of Thoughts) 검증 프로세스 규칙에서 기존 C/C++ 소스/헤더 분류 체계 외에 Lua(.lua) 스크립트에 대한 명시적인 구문 및 구조 검증 규칙(LUA correctness checklist)을 추가하여 시니어가 단순한 Lua 함수를 C++ 시각으로 빈 함수 혹은 C++ 중괄호 본문 누락으로 취급하지 않도록 함.
- 또한, 시니어가 단순한 스타일 취향 차이(예: 단순화된 함수 반환값, 플레이스홀더를 채운 간단한 반환 등)로 인해 정상적으로 기능하는 코드를 집요하게 거부(KISS 원칙 오해 및 오적용)하여 무한 리워크 루프 및 작업 실패(Boss Board 행)가 발생하는 현상을 방지하고자, 검증 단계(Correctness Verification)를 통과하고 심각한 오류가 없는 한 승인(APPROVE)하도록 스타일 거절 가이드를 대폭 완화함.
