# 📄 AXON 설계 원칙 — 단순 → 필연적 복잡 (v0.0.17)

## 🎯 목적
AXON 시스템 확장 시, 불필요한 복잡성을 방지하고  
필요한 시점에만 기능을 추가하기 위한 기준 정의

---

## 🧠 핵심 원칙

- 단순함은 출발점이다
- 복잡성은 “필요할 때만” 추가한다
- 모든 기능은 이유가 있어야 한다

---

## ⚙️ 의사결정 기준

### 1. 이 기능이 없으면 시스템이 깨지는가?
- YES → 추가
- NO → 보류

### 2. 기존 구조를 깨는가?
- YES → 재설계
- NO → 진행

### 3. 필수인가, 편의인가?
- 필수 → 지금
- 편의 → 나중

---

## 🚫 안티 패턴

기능을 계속 쌓기만 하는 구조:

기능 추가 → 또 추가 → 또 추가  
→ 시스템 이해 불가  
→ 유지보수 불가

---

## ✅ 올바른 흐름

문제 발생  
→ 원인 분석  
→ 최소 해결  
→ 검증  
→ 다음 단계

---

## 📊 현재 AXON 상태

- Execution
- Retry / Fallback
- Timeout

- Multi Worker
- Observability
- Persistence
- API

👉 이미 운영 가능한 구조
