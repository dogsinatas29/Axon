# SYNAPSE Age Calculator - Executable Architecture (v0.4)

## 🎯 PURPOSE

This document defines a **directly executable architecture** based
strictly on GEMINI.md. No abstraction, only implementation-ready
structure.

------------------------------------------------------------------------

# 📦 PROJECT STRUCTURE

    project/
    │
    ├── main.py
    ├── database.py
    ├── calculator.py
    ├── validators.py
    │
    ├── input_year.py
    ├── input_month.py
    ├── input_day.py
    │
    └── requirements.txt

------------------------------------------------------------------------

# 🔗 MODULE RESPONSIBILITIES

## 1. main.py (ENTRYPOINT)

### Role:

-   System orchestrator
-   Controls full execution flow

### Flow:

1.  Get user name
2.  Check DB (bypass logic)
3.  If not found → input flow
4.  Validate inputs
5.  Calculate age
6.  Store/update DB
7.  Display result

------------------------------------------------------------------------

## 2. database.py

### Role:

-   SQLite3 handling
-   Data persistence
-   pandas integration (optional)

### Interface:

``` python
def get_user(name: str) -> dict | None
def save_user(data: dict) -> None
```

------------------------------------------------------------------------

## 3. calculator.py

### Role:

-   Age calculation logic

### Dependency:

-   python-dateutil

### Interface:

``` python
def calculate_age(year: int, month: int, day: int) -> dict
```

Return:

``` python
{
    "korean_age": int,
    "international_age": int
}
```

------------------------------------------------------------------------

## 4. validators.py

### Role:

-   Central validation logic

### Interface:

``` python
def validate_year(year: int) -> int
def validate_month(month: int) -> int
def validate_day(year: int, month: int, day: int) -> int
```

------------------------------------------------------------------------

## 5. INPUT LAYER

### Files:

-   input_year.py
-   input_month.py
-   input_day.py

### Role:

-   User input handling
-   Loop until valid

### Interface:

``` python
def input_year() -> int
def input_month() -> int
def input_day(year: int, month: int) -> int
```

------------------------------------------------------------------------

# 🔄 EXECUTION FLOW

    main.py
      ↓
    database.get_user()
      ↓
    [IF EXISTS] → bypass → calculator
    [ELSE]
      ↓
    input_year → validate_year
    input_month → validate_month
    input_day → validate_day
      ↓
    calculator.calculate_age()
      ↓
    database.save_user()
      ↓
    output (rich)

------------------------------------------------------------------------

# 🧪 EXECUTION REQUIREMENTS

System must:

-   Run via `python main.py`
-   Accept user input
-   Validate all inputs
-   Calculate correct age
-   Persist data in SQLite
-   Display output via rich

------------------------------------------------------------------------

# 🔒 DESIGN RULES

-   No cross-module logic leakage
-   All validation via validators.py
-   All DB access via database.py
-   main.py is the only orchestrator

------------------------------------------------------------------------

# 🚀 RESULT

This architecture guarantees:

-   Direct executability
-   Clear module boundaries
-   Deterministic flow
-   No abstraction leakage
