# 🚀 Implementation Plan - v0.0.14 Intelligence Factory Bootstrapping

---

## 🏛️ Project Goal
**"터미널 대화형 인터페이스를 통한 공장 가동 및 아키텍트 주도의 자동 설계서(Architecture.md) 작성 구현."**

---

## 📅 작업 단계 (Phases)

### Phase 1: 인터랙티브 부트스트래핑 (✅ Completed)
1. **Model Discovery**: 환경 변수에서 사용 가능한 LLM API 키(Gemini, Claude, GPT) 스캔 로직 구현.
2. **HR Assignment Loop**:
   - `main.rs`에 아키텍트, 시니어, 주니어의 머릿수와 지능을 선택하는 대화형 CLI 구현.
   - `Daemon` 구조체를 확장하여 역할별 독립적인 `ModelDriver`를 수용하도록 리팩토링.
3. **Spec Path Selection**: 작업의 근거가 될 명세서 파일 경로를 사용자로부터 입력받음.

### Phase 2: 아키텍트의 설계서(Architecture.md) 작성 (🏗️ Next)
1. **Spec Data Ingestion**: `main.rs`에서 받은 `spec_path`의 내용을 데몬의 초기 `architecture_guide`로 주입.
2. **Architect Setup Task**:
   - 아키텍트 전용 메서드 `bootstrap_architecture` 구현.
   - 아키텍트가 명세서를 읽고, 시스템 전체 구조와 섹션을 정의한 `architecture.md`를 자동으로 생성 및 저장.
3. **SSOT 초기화**: 생성된 설계서를 데몬의 전역 지침(Architecture Guide)으로 확정.

### Phase 3: 통합 데몬 오케스트레이션 및 병렬 가동
1. **Task Slicing**: 아키텍트가 작성한 `architecture.md`의 섹션별로 작업을 분해하여 `Dispatcher`에 자동 등록.
2. **Role-based Dispatching**: 각 작업이 지정된 역할(Junior 구현 -> Senior 검수 -> Architect 검증)에 따른 전용 모델로 처리되는지 확인.
3. **Unified Logging**: 단일 터미널에서 모든 병렬 에이전트의 작업 로그를 투명하게 표시.
