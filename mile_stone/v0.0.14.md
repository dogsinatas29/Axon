### 🚨 긴급 장애 분석 (Emergency Triage: Greedy Parser, Context Hijacking & UI Disconnect)
1. **러스트 파서의 '탐욕적' 버그 (Greedy Parsing Fix)**
   - 현상: `rfind`를 사용해 문서 맨 끝의 JSON 블록 닫는 백틱까지 통째로 긁어와 마크다운과 JSON이 비빔밥처럼 섞이는 현상 발생.
   - 해결: `find`와 문자열 슬라이싱을 통해 정확히 마크다운 블록만 정밀 절개(`axon-daemon/src/lib.rs`).
2. **컨텍스트 하이재킹 (Domain Dominance Override)**
   - 현상: `GEMINI.md`의 강력한 도메인 명세(ECS 엔진 등)에 짓눌려, 아키텍트가 프로토콜을 무시하고 기존 구조를 맹신함.
   - 해결: 프롬프트에 `--- CRITICAL PROTOCOL ENFORCEMENT ---` 블록을 신설, 기존 시스템을 무조건 'Node' 레벨로 수직 격하시키고 'Hub'를 씌우도록 위계 질서 확립.
3. **UI 백엔드 미구현 및 API 쿼터 고갈 한계**
   - 워크 보드 텅 빈 현상: 하드코딩된 `/api/tasks -> vec![]` 가 원인 (백엔드 연동 미완성).
   - 인사 관리 텅 빈 현상: 터미널 채용이 콘솔 프린트만 하고 DB Insert(`save_agent`) 로직을 누락함.
   - 시그널(Signals) 먹통 현상: 코딩을 마치기 전 무료 API 쿼터 한도 초과(`Quota Error`)로 에이전트가 중도 폭사하여, 웹소켓 패킷(`event_bus.publish`)이 발송되지 못함.
4. **불필요한 재부팅 시 토큰 낭비 방지 (Fast Resume)**
   - 현상: 데몬을 껐다 켰을 때, 이전에 만들어진 `architecture.md`가 있음에도 무조건 처음부터 LLM을 파크업하여 귀중한 쿼터/토큰을 낭비함.
   - 해결: 워크스페이스 내에 `architecture.md`가 감지되면, `[1] Resume` 옵션을 통해 즉각적으로 백엔드 서버(localhost)만 띄우고 과거 데이터를 읽어들이도록 분기문 삽입.
