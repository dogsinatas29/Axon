# 🚀 Implementation Plan: v0.0.15 - AI Union Management (Quota Handling & Rate Limiter)

## 📌 개요
본 계획서는 **v0.0.15.md** 마일스톤에 정의된 "전역 속도 조율기(Throttle)"와 "에러/쿼터 대기열 핸들링" 그리고 "프론트엔드 상태 동기화"를 구현하기 위한 개발 타임라인과 체크리스트입니다.

---

## 🛠️ Phase 1: 모델 계층 쿼터 감지 및 Retry (Model Driver)
*가장 하위 단위인 API 발송 단에서 429 에러를 감지하고 스스로 복구할 수 있는 생명력을 부여합니다.*

- [ ] **에러 타입 정밀 파싱 로직 추가**
  - 타겟 파일: `crates/axon-model/src/lib.rs` (Gemini 드라이버 내부)
  - 내용: HTTP `429` 혹은 에러 메시지 텍스트(`Quota exceeded`)를 정규식/스트링 매칭으로 파싱.
  - 내용: `Please retry in X.Xs` 문자열에서 대기 시간(N초)을 추출하는 로직 구현.
- [ ] **자동 Backoff 및 재시도 루프(Retry Loop) 장착**
  - 타겟 파일: `crates/axon-model/src/lib.rs`
  - 내용: Quota 에러 감지 시 즉시 에러(`Err`)를 위로 뱉어버리는 대신, 해당 스레드를 멈춰두고 `tokio::time::sleep`으로 대기. 대기 완료 후 동일한 인풋으로 반복 요청 재시도(최대 3~5회).

## 🛠️ Phase 2: 스케줄러 전역 속도 조율기 (Global Rate Limiter)
*여러 주니어가 동시에 API를 때려 쿼터 에러 자체가 발생하지 않도록 근본적인 예방 백신을 꽂습니다.*

- [ ] **동적 에이전트 간격 조율 (Semaphore / Interval)**
  - 타겟 파일: `crates/axon-daemon/src/lib.rs` (에이전트 할당 혹은 스레드 병렬 실행 루프)
  - 내용: Free Tier(15 RPM = 4초당 1발) 한계를 지키기 위해, `tokio::time::interval`이나 `Semaphore`를 도입.
  - 내용: 동시에 N명의 에이전트가 돌더라도 실제 구글 API 서버로 나가는 패킷은 4초의 텀을 두고 순차적으로 빠져나가게끔 `await` 지연 분배 체계 구현.

## 🛠️ Phase 3: 데몬 & 프론트엔드 연동 상태 전파 (State Propagator)
*구글의 심사가 까다로워 대기 중일 때, 사용자가 빈 화면을 보며 당황하지 않게끔 친절한 안내판을 띄웁니다.*

- [ ] **데이터베이스 및 상태 정의 업데이트**
  - 타겟 파일: `crates/axon-core/src/lib.rs`
  - 내용: 기존의 `TaskState` 혹은 `ThreadState`에 `WaitingQuota(u64)`라는 열거형 상태 추가. (초 단위 타이머 포함)
- [ ] **웹 소켓 시그널 송신 및 DB 반영**
  - 타겟 파일: `crates/axon-daemon/src/lib.rs`
  - 내용: 모델 드라이버가 429 에러를 맞고 Sleep에 들어가기 직전, 데몬에게 이벤트를 전파. デーモン은 DB 상태를 업데이트하고 프론트엔드 관제실로 WebSocket `QUOTA_EXCEEDED` 이벤트(페이로드: 대기 시간) 전송.
- [ ] **Studio UI 대기 타이머 시각화**
  - 타겟: `Studio UI` 프론트엔드 모달.
  - 내용: "Analyzing transmission history..." 빈 화면 대신 "⚠️ 구글 API 무료 쿼터 제한 도달. N초 후 재개됩니다." 경고 뱃지 렌더링.

## 🛠️ Phase 6: 사장님 전용 🎨 GitHub-Style Advanced Diff UI (WorkBoard)
*게시판의 단편적인 패치 글자를 깃허브의 'Files Changed' 탭 수준으로 끌어올려, 대규모 코드를 줄바꿈 없이 쏟아내도록 혁신합니다.*

- [ ] **Advanced Diff Parser 구현**
  - 타겟: `ThreadDetail.tsx` (Frontend)
  - 내용: `@@` 메타데이터를 파싱하여 `+` (추가), `-` (삭제), ` ` (컨텍스트) 라인을 배열로 분리.
- [ ] **Visual Diff 렌더링 시스템 구축**
  - 내용: 
    - 추가된 줄: 초록색 배경(rgba(0, 255, 0, 0.1)) + 녹색 폰트.
    - 삭제된 줄: 빨간색 배경(rgba(255, 0, 0, 0.1)) + 빨간 폰트.
    - 줄 번호 하이라이트: 깃허브와 동일한 좌측 여백 레이아웃 적용.
- [ ] **"The GitHub Flow" UI 스타일링**
  - 내용: 불필요한 줄바꿈이나 여백 마진을 극도로 줄여서, 수백 줄의 코드도GitHub 화면처럼 막힘없이 쭉 써내려가듯 표출.

## 📅 후속 우선 작업
## 🛠️ Phase 7: Strict Priority Concurrency Scheduling (Quota Hell 방지)
*다수의 에이전트가 API 쿼터를 동시에 소모하지 않도록 '병렬 농도'를 강제 조절합니다.*

- [ ] **전역 세마포어(Semaphore) 도입**
  - 타겟: `crates/axon-daemon/src/lib.rs` (`run` 루프)
  - 내용: 동시에 실행되는 `handle_assignment` 비동기 태스크 수를 최대 3개로 제한하는 `Arc<Semaphore>` 적용.
- [ ] **태스크 집중 가중치 시스템**
  - 내용: 한 태스크가 시작되면 '주니어 제출 -> 시니어 리뷰 -> 아키텍트 검증'이 끝날 때까지 해당 슬롯을 점유하게 하여, 한 번에 완결된 결과(Diff)가 빠르게 나오도록 유도.
- [ ] **Sequential Worker Dispatch**
  - 내용: 17개 태스크를 한꺼번에 할당하지 않고, 큐(Queue)에서 순차적으로 한 개씩 처리되는 '1인 생산 라인' 모드 지원.

## 🛠️ Phase 8: OS Native Locale Injection (지능형 현지화)
*사용자의 OS 환경에 맞춘 자동 언어 통일로 게시판 가독성을 극대화합니다.*

- [ ] **시스템 로케일(Locale) 탐지 엔진 구현**
  - 타겟: `crates/axon-daemon/src/main.rs`
  - 내용: `std::env::var("LANG")` 등을 통해 현재 OS 언어(ko_KR, en_US 등) 추출 및 Daemon 상태 저장.
- [ ] **에이전트 페르소나 언어 강제 주입**
  - 타겟: `crates/axon-agent/src/lib.rs` (Persona Generator)
  - 내용: 탐지된 Locale 값을 프롬프트에 동적 삽입하여 모든 Task Output 언어를 고정.
- [ ] **기존 영문 태스크 자동 번역 리팩토링**
  - 내용: 부트스트랩 시 기존 DB에 영문 태스크가 있을 경우 아키텍트에게 "사장님 언어"로 제목/설명을 변환하라는 특별 지시 하달.
