## 🧠 상세 설계 및 로직

### 1. 🚦 전역 속도 조율기 (Global Rate Limiter / Throttle)
- **설계 의도**: 다수의 주니어 에이전트가 동시에 깨어나 구글 API에 무결재 돌격하는 것을 방지.
- **구현 방식**: 
  - `axon_daemon` 내부의 작업 할당 스케줄러에 Token Bucket 또는 시간 기반 (예: 4초당 1건) 제동 장치(Throttle) 추가.
  - 제미나이 Free Tier의 제한인 15 RPM(Requests Per Minute)을 선제적으로 초과하지 않도록, 에이전트들의 API 호출 대기열(Queue)을 동적으로 제어.

### 2. 🛡️ 429 Quota Error 핸들러 및 자동 재개 (Retry Backoff)
- **설계 의도**: 만약 쿼터 초과(`HTTP 429 Too Many Requests`)를 맞더라도 에이전트가 패닉에 빠지지 않고 유연하게 대처.
- **구현 방식**: 
  - `axon_model` 모델 드라이버 계층에서 에러 응답(`Err`)을 파싱하여, Quota 에러일 경우 실패 처리를 하지 않음.
  - 에이전트 상태를 일시 중지(`WAITING_QUOTA`) 상태로 전환하고 구글이 제시한 대기 시간(예: 50초) 혹은 지수 백오프(Exponential Backoff)만큼 Sleep(대기) 수행.
  - 쿨타임 종료 후, 들고 있던 프롬프트를 정확히 동일한 시점부터 다시 전송(Resume).

### 3. 🚨 시각적 에러 시그널링 (Web UI WebSocket)
- **설계 의도**: 파업 중인(?) 주니어들의 상태를 관제탑 메뉴얼과 스레드 창에 명확히 표출.
- **구현 방식**:
  - 백엔드에서 429 에러가 발생한 순간, WebSocket을 통해 프론트엔드(`Studio UI`)에 `QUOTA_EXCEEDED` 패킷 전송.
  - 프론트엔드는 텅 빈 "Analyzing transmission history..." 화면 대신, 빨간색 경고 배지와 함께 **"구글 쿼터 제한 대기 중... (N초 후 재시도)"** 라는 타이머를 스레드 모달 내부에 표출.

## ⚠️ 예외 처리 및 주의 사항
- 쿼터 대기 중일 때 유저가 데몬을 끄더라도, 작업 정보(`axon.db`)가 유실되지 않도록 Task Progress 처리를 원자적(Atomic)으로 보장할 것.
- Rate Limiter가 너무 엄격하게 적용되어 단일 호출마저 지연되지 않게끔 버스트(Burst) 임계값 적절히 튜닝.
