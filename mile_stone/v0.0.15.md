## 🧠 상세 설계 및 로직

### 1. 🚦 전역 속도 조율기 (Global Rate Limiter / Throttle)
- **설계 의도**: 다수의 주니어 에이전트가 동시에 깨어나 구글 API에 무결재 돌격하는 것을 방지.
- **구현 방식**: 
  - `axon_daemon` 내부의 작업 할당 스케줄러에 시간 기반 (4.1초당 1건) 제동 장치 추가. [✅ Completed]

### 2. 🛡️ 429 Quota Error 핸들러 및 자동 재개 (Retry Backoff)
- **설계 의도**: 만약 쿼터 초과를 맞더라도 에이전트가 패닉에 빠지지 않고 유연하게 대처.
- **구현 방식**: 
  - `axon_model`에서 `QUOTA_WAIT:N` 에러를 반환하고, `AgentRuntime`이 이를 가로채 `SystemWarning` 이벤트 발행 후 대기 및 재시도. [✅ Completed]

### 3. 🚨 시각적 에러 시그널링 (Web UI WebSocket)
- **설계 의도**: 파업 중인(?) 주니어들의 상태를 관제탑 메뉴얼과 스레드 창에 명확히 표출.
- **구현 방식**:
  - `EventBus`를 통해 `SystemWarning` 이벤트를 UI 실시간 시그널 탭으로 전송. [✅ Completed]

### 4. ♻️ 태스크 자동 복구 (Resume Task Recovery)
- **설계 의도**: `--resume` 시 DB에 저장된 완료되지 않은 태스크들이 멈춰있지 않고 즉시 재개되도록 함.
- **구현 방식**:
  - `Daemon::run` 시작 시 DB에서 `Pending/InProgress` 상태의 태스크를 전역 큐에 재진입시킴. [✅ Completed]

### 5. 👁️ Studio UI 코드 가독성 개선 (Unescape)
- **설계 의도**: 주니어가 제출한 코드의 줄바꿈(`\n`), 탭(`\t`)이 그대로 글자로 노출되는 현상을 해결.
- **구현 방식**:
  - `ThreadDetail.tsx`의 텍스트 렌더링 로직에 `Unescape` 처리 적용. [✅ Completed]

### 6. 🎨 GitHub-Style Advanced Diff Rendering (WorkBoard)
- **설계 의도**: 게시판(Board)에서의 코드 가독성을 GitHub의 'Pull Request' 탭 수준으로 끌어올림.
- **구현 방식**: 
  - `@@` 패치 데이터 파싱 및 줄 번호/하일라이팅 처리. [✅ Completed]

### 7. 🚦 Strict Priority Concurrency Scheduling (Quota Hell 방지)
- **설계 의도**: 동시에 너무 많은 주니어(17명 등)가 깨어나 API 쿼터를 조기 소모하는 '쿼터 지옥' 현상을 원천 차단.
- **구현 방식**: 전역 세마포어(Semaphore) 도입 및 최대 활성 태스크 수 3개 제한. [🏗️ Planned]

### 8. 🌐 OS Native Locale Injection (지능형 현지화)
- **설계 의도**: 사용자가 설치한 실제 OS의 언어 설정(Locale)을 자동 감지하여, 모든 에이전트의 작업 결과물(태스크 제목, 설명, 계획서)을 해당 언어로 생성함. 영문으로 된 '읽기 힘든' 게시판 문제를 원천 차단.
- **구현 방식**:
  - `axon_daemon` 구동 시 `std::env::var("LANG")` 또는 `sys-locale` 라이브러리를 통해 시스템 언어 추출.
  - 추출된 Locale 정보를 모든 에이전트의 **System Persona**에 강제 주입: *"모든 태스크 명칭과 설명은 [Detected Locale] 언어로 작성하라."*
  - 기존 영문 태스크를 발견할 경우, 아키텍트가 이를 해당 Locale로 자동 번역 및 재작성 유도. [🏗️ Planned]
