# 📄 AXON v0.0.17 Execution Harness Spec

## 🎯 Purpose

Ensure LLM strictly follows the specification in a controlled,
step-by-step validated execution flow.

------------------------------------------------------------------------

## 1. 🧱 Bootstrap Harness Structure

### Flow

    [Phase 0] Load Spec Index
    [Phase 1] Load Section
    [Phase 2] Assertion Check
    [Phase 3] Unlock Next Phase

### Principle

-   No skipping
-   No implicit understanding
-   Explicit validation required

------------------------------------------------------------------------

## 2. 📑 Spec Index

### Phase Order

1.  Configuration
2.  Validation
3.  Bootstrapping
4.  Execution
5.  Observability

### Rules

-   Must follow order strictly
-   Each phase must be completed before moving forward

------------------------------------------------------------------------

## 3. 🔒 Phase Lock System

### Assertion Format

    [ASSERTION]
    - What is the role of this phase?
    - What output does it produce?
    - What is passed to the next phase?

### Unlock Condition

-   Structured answer
-   Key concepts present

------------------------------------------------------------------------

## 4. 🧪 Execution Contract

### Example

    [CONTRACT]

    Input:
    - AgentConfig

    Output:
    - ValidatedAgentRegistry

    Invariant:
    - All agents must have runtime
    - All models must be resolvable

------------------------------------------------------------------------

## 5. 📊 State Transition

    CONFIG_LOADED
    → VALIDATED
    → BOOTSTRAPPED
    → RUNNING
    → PAUSED

------------------------------------------------------------------------

## 6. 🧠 Context Injection

    [CONTEXT BLOCK]

    Agents:
    - role
    - model
    - runtime

    Constraints:
    - sampling_rate
    - queue_limit

    Warnings:
    - validation results

------------------------------------------------------------------------

## 7. 🔁 Retry / Failure Policy

    Retry:
    - max_retry = 3

    Failure:
    - soft fail → skip
    - hard fail → escalate

------------------------------------------------------------------------

## 8. 🧩 Minimal Interfaces

``` rust
trait Runtime {
    async fn execute(task: Task) -> Result<Output>;
}

trait Validator {
    async fn validate(agent: AgentConfig) -> ValidationResult;
}
```

------------------------------------------------------------------------

## 9. 🚫 What NOT to Do

    AXON does NOT:
    - Auto assign roles
    - Auto select models
    - Auto scale agents

------------------------------------------------------------------------

## ✅ Summary

    Spec → MUST be followed
    Execution → MUST be validated
    Progress → MUST be locked step-by-step
