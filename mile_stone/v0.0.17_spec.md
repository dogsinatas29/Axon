📄 AXON v0.0.17 Spec
User-Controlled Local/Cloud Orchestration
0. 🎯 Goal (정확한 목표 정의)
Primary Goal

사용자가 선택한 로컬/클라우드 LLM 구성을 기반으로
검증 가능하고 예측 가능한 방식으로 실행되는 오케스트레이션 시스템 구축

Success Criteria
사용자가 지정한 LLM 구성 그대로 실행됨 (자동 변경 없음)
잘못된 설정에 대해 명확한 검증 및 경고 제공
로컬/클라우드 구성이 동일한 인터페이스로 동작
병목 상황에서도 시스템이 멈추지 않고 제어 가능 상태 유지
실행 상태를 사용자가 실시간으로 이해 가능 (Observability 확보)
1. 🧠 Core Concept
기존 (폐기)
LLM = 자동으로 배치되는 분산 노드
v0.17
LLM = 사용자가 명시적으로 배치한 실행 에이전트
핵심 정의

AXON은 결정하지 않는다. 실행하고 검증한다.

4. 🔍 Phase 1 — Validation (Discovery 축소)
Goal

사용자가 지정한 LLM이 실제로 동작 가능한지 검증

Local Validation
GET /api/tags

검증 항목:

endpoint reachable
model 존재 여부
Cloud Validation
API Key 존재 여부
간단 ping 테스트
결과
[OK] mistral:latest @ 127.0.0.1
[WARN] model not found
[FAIL] endpoint unreachable
정책
실패 시 실행 중단 여부는 사용자 선택
자동 대체 없음

8. ⚙️ Execution Engine (핵심 유지)
기본 흐름
Task 생성
    ↓
Junior 실행
    ↓
Senior 검토
    ↓
Architect 검증
    ↓
Commit
Agent 선택
round_robin(role_agents)
Runtime 실행
match agent.runtime {
    Local => local_runtime.execute(),
    Cloud => cloud_runtime.execute(),
}
9. 🔁 Progressive Validation (수정)
목표
무한 reject loop 방지
병목 제어
구성
1. Sampling Review
if random() < sampling_rate {
    review()
}
2. Retry Limit
if retry_count > 3 {
    escalate_to_architect()
}
3. Cross Validation (옵션)
Local ↔ Cloud 교차 검증
10. 🚦 Bottleneck Control
핵심 정책

자동 확장 ❌
속도 제어 ✅

(A) Queue Limit
if review_queue.len() > limit {
    pause_junior_dispatch();
}
(B) Adaptive Throttle
delay = last_latency * 1.2
(C) Resume
if queue_recovered {
    resume_dispatch();
}
11. 🔌 Failure Handling
요구사항
노드 장애 시 해당 agent만 제외
시스템 전체는 계속 실행
if agent_unavailable {
    mark_unavailable();
    skip_agent();
}
특징
자동 대체 없음 (fallback_enabled일 때만 예외)
