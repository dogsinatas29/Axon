# AXON v0.0.18 Formatting Stabilization (AST → Code)

## 🎯 PURPOSE

Minimize code degradation when converting AST back to source code.

------------------------------------------------------------------------

# 1. PROBLEM

AST-based merge causes:

-   Comment loss
-   Formatting drift
-   Import/order changes

Result: Code works but becomes untrustworthy and hard to maintain.

------------------------------------------------------------------------

# 2. CORE STRATEGY

Do NOT fully regenerate code from AST.

Instead:

ORIGINAL CODE (CURRENT) + AST-based modifications → Partial replacement

------------------------------------------------------------------------

# 3. HYBRID APPROACH

## Concept

Preserve untouched code, replace only modified functions.

Example:

``` python
# preserved
def untouched():
    pass

# replaced
def updated():
    return 42
```

------------------------------------------------------------------------

# 4. PARTIAL REPLACEMENT

## Step 1: Locate function span

``` python
def find_function_span(source, func_name):
    # find start/end lines
    pass
```

------------------------------------------------------------------------

## Step 2: Replace only that block

-   Keep rest of file intact
-   Maintain comments and structure

------------------------------------------------------------------------

# 5. FORMATTER (REQUIRED)

Apply formatter after merge:

-   Black (recommended)
-   Ruff (optional)

------------------------------------------------------------------------

## Pipeline Position

merge → partial replace → format → commit

------------------------------------------------------------------------

# 6. RULES

-   NEVER regenerate full file from AST
-   ALWAYS preserve original structure when possible
-   ONLY modify minimal necessary region

------------------------------------------------------------------------

# 7. MVP SCOPE

-   Function-level replacement
-   Black formatter integration
-   Comment preservation

------------------------------------------------------------------------

# 8. LIMITATIONS

-   Class/method not handled yet
-   Exact formatting may slightly shift
-   Complex nested structures need future work

------------------------------------------------------------------------

# 9. FUTURE EXTENSIONS

-   Token-level mapping (AST + tokenize)
-   Comment-aware AST merge
-   Class-level support
-   Import block stabilization

------------------------------------------------------------------------

# 🚀 RESULT

-   Stable readable code
-   Minimal diff noise
-   Developer trust preserved

👉 Enables practical AST-based evolution
