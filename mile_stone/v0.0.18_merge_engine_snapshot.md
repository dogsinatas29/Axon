# AXON v0.0.18 Merge Engine + Snapshot Design

## 🎯 PURPOSE

Define a practical merge engine implementation and a snapshot system
built on top of it.

------------------------------------------------------------------------

# 1️⃣ MERGE ENGINE (MVP IMPLEMENTATION)

## Goal

Perform safe 3-way merge using BASE, CURRENT, NEW

------------------------------------------------------------------------

## 1.1 Core Strategy

-   File-level merge first
-   Line-based diff (difflib)
-   Conflict detection (fail-fast)

------------------------------------------------------------------------

## 1.2 Implementation (MVP)

``` python
import difflib

def merge_lines(base, current, new):
    if current == base:
        return new

    if new == base:
        return current

    if current == new:
        return current

    # fallback: detect conflict
    return None
```

------------------------------------------------------------------------

## 1.3 File Merge

``` python
def merge_file(base, current, new):
    result = merge_lines(base, current, new)

    if result is None:
        return "CONFLICT"

    return result
```

------------------------------------------------------------------------

## 1.4 Multi-file Merge

``` python
def merge_all(base_map, current_map, new_map):
    merged = {}

    all_files = set(base_map) | set(current_map) | set(new_map)

    for f in all_files:
        base = base_map.get(f, "")
        current = current_map.get(f, "")
        new = new_map.get(f, "")

        result = merge_file(base, current, new)

        if result == "CONFLICT":
            return None

        merged[f] = result

    return merged
```

------------------------------------------------------------------------

## 1.5 Behavior

-   Safe overwrite when possible
-   Conflict triggers LLM loop
-   No partial merge allowed

# 2️⃣ SNAPSHOT SYSTEM

## Goal

Provide BASE state for future merges

------------------------------------------------------------------------

## 2.1 Snapshot Requirement

3-way merge requires:

BASE = last successful commit

------------------------------------------------------------------------

## 2.2 Storage Strategy

``` text
.axon/
    snapshot.json
```

------------------------------------------------------------------------

## 2.3 Snapshot Structure

``` json
{
  "main.py": "file content or hash",
  "utils.py": "file content or hash"
}
```

------------------------------------------------------------------------

## 2.4 Save Snapshot

``` python
import json

def save_snapshot(files, path):
    data = {name: code for name, code in files}
    with open(path, "w", encoding="utf-8") as f:
        json.dump(data, f)
```

------------------------------------------------------------------------

## 2.5 Load Snapshot

``` python
def load_snapshot(path):
    try:
        with open(path, "r", encoding="utf-8") as f:
            return json.load(f)
    except:
        return {}
```

------------------------------------------------------------------------

# 3️⃣ PIPELINE INTEGRATION

``` python
def process(new_files, current_files, snapshot):
    merged = merge_all(snapshot, current_files, new_files)

    if merged is None:
        return "CONFLICT"

    atomic_commit(list(merged.items()), "./output")

    save_snapshot(merged, ".axon/snapshot.json")

    return "SUCCESS"
```

------------------------------------------------------------------------

# 4️⃣ DESIGN DECISIONS

### Why content snapshot (not hash only)?

-   Merge requires actual content comparison
-   Hash-only insufficient for diff

------------------------------------------------------------------------

### Why file-level first?

-   Simpler
-   Deterministic
-   Expandable to function-level later

------------------------------------------------------------------------

# 5️⃣ LIMITATIONS (INTENTIONAL)

-   No fine-grained merge
-   Conflict = fail-fast
-   No semantic merge yet

------------------------------------------------------------------------

# 6️⃣ RESULT

This system enables:

-   Stable 3-way merge
-   Reliable base tracking
-   Safe iterative evolution
