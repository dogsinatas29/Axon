# 1. MULTI-FILE HEURISTIC PARSER

## Problem

LLM may output multiple files as unordered code blocks.

## Solution

Treat each code block as a file candidate.

------------------------------------------------------------------------

## 1.1 Parsing Strategy

``` python
def parse_heuristic_multi(output, context):
    blocks = extract_code_blocks(output)

    candidates = []

    for block in blocks:
        filename = infer_filename(block, context)
        candidates.append((filename, block))

    return resolve_conflicts(candidates)
```

------------------------------------------------------------------------

## 1.2 Block Classification

``` python
def classify_block(code):
    if "class " in code or "def " in code or "import " in code:
        return "module"
    return "unknown"
```

------------------------------------------------------------------------

## 1.3 Merge Same File Blocks

``` python
def merge_same_file(candidates):
    merged = {}

    for name, code in candidates:
        if name not in merged:
            merged[name] = code
        else:
            merged[name] += "\n\n" + code

    return list(merged.items())
```

------------------------------------------------------------------------

## 1.4 Conflict Resolution

``` python
def resolve_conflicts(candidates):
    merged = merge_same_file(candidates)

    unique = {}
    for name, code in merged:
        name = ensure_unique(name, unique)
        unique[name] = code

    return list(unique.items())
```

# 2. DEPENDENCY-BASED FILENAME RE-INFERENCE

## Problem

Incorrect filename breaks imports.

------------------------------------------------------------------------

## 2.1 Import Extraction

``` python
import re

def extract_imports(code):
    return re.findall(r"from (\w+) import|import (\w+)", code)
```

------------------------------------------------------------------------

## 2.2 Missing Module Detection

``` python
def find_missing_modules(files):
    names = {f.replace(".py", "") for f, _ in files}
    missing = set()

    for _, code in files:
        imports = extract_imports(code)

        for a, b in imports:
            mod = a or b
            if mod and mod not in names:
                missing.add(mod)

    return missing
```

------------------------------------------------------------------------

## 2.3 Filename Correction

``` python
def fix_filenames(files):
    missing = find_missing_modules(files)

    updated = []

    for name, code in files:
        for mod in missing:
            if mod in code.lower():
                name = f"{mod}.py"
        updated.append((name, code))

    return updated
```

------------------------------------------------------------------------

## 2.4 Retry Loop

``` python
for _ in range(3):
    files = parse_heuristic_multi(output, context)
    files = fix_filenames(files)

    if validate_dependencies(files):
        break
```

LLM OUTPUT\

# 4. DESIGN RULES

-   Filename inference before merge
-   Dependency validation before commit
-   Retry limited (max 3)
-   No commit on invalid dependency state

------------------------------------------------------------------------

# 5. RISKS & MITIGATION

## Risk: Duplicate files

## Risk: Circular dependency
