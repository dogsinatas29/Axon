# 🧠 AXON v0.0.18 SPEC (Output Contract 기반 코드 생성)

## 🎯 GOAL
승인(Approved) 상태에 도달한 작업은 반드시:
- 소스 코드 생성
- 파일 생성
- CLOSE 상태 전이
까지 완료되어야 한다.

---

## 🚨 문제 정의
현재 시스템 문제:
- LLM 출력 비결정성
- 코드 대신 설명 출력
- 파서 실패
- 결과적으로 src 미생성

👉 승인 이후에도 실제 결과물이 없음

---

## 🧩 핵심 전략
LLM을 자유 생성기가 아니라 **Output Contract 기반 실행 엔진**으로 강제

---

## 📦 OUTPUT CONTRACT

[OUTPUT]

FILE: <filename>

```python
<code>
```

END_FILE

### ❗ 금지 사항
- 설명 텍스트 금지
- reasoning 금지
- 코드 블록 외 텍스트 금지

### 필수 조건
- FILE 블록 최소 1개
- filename 중복 금지
- [OUTPUT] 1회만 허용

---

## 🔒 파서 규칙

### Phase 1: Structure Extraction
- FILE 단위 분리
- 코드 블록 추출 (case-insensitive)
- ```python / ```Python / ``` 허용

### Phase 2: Validation
- filename 존재
- 코드 non-empty
- filename sanitize (.. 금지)

---

## ❌ 실패 조건
- FILE 없음
- 코드 블록 없음
- 빈 코드
- filename 없음

---

## 🔁 Retry 전략

1. Format Fail → FORMAT VIOLATION 재시도
2. Code Fail → CODE INVALID 재시도
3. Empty → temperature↓ / tokens↑

---

## ⚙️ 파이프라인

Junior (CODE ONLY)
→ Format Validation
→ Code Validation
→ Senior Review
→ Architect Approval
→ Commit
→ CLOSE

---

## 🧪 Code Validation

```python
def validate_code(code):
    try:
        compile(code, "<string>", "exec")
        return True
    except:
        return False
```

---

## 📁 Commit (Atomic)

```python
tmp_dir = create_temp_dir()
write_all_files(tmp_dir)

if success:
    move_to_target(tmp_dir)
else:
    rollback(tmp_dir)
```

## 📊 상태 전이

→ RUNNING

→ GENERATING_FILES

→ FILE_CREATED
→ CLOSED
