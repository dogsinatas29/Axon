# AXON v0.0.18 Formatting Stabilization Phase

## PURPOSE

Stabilize code output after AST transformations.

## CORE RULES

-   Never regenerate full file
-   Replace only modified spans
-   Preserve comments and layout

## PIPELINE

AST Merge → Span Replace → Formatter → Commit

## REQUIREMENTS

-   Function-level replacement
-   Black formatter integration
-   Comment preservation

## RESULT

Readable, stable, low-noise diffs
