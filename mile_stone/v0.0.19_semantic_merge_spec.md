# AXON v0.0.19 SEMANTIC MERGE SPEC (IMPLEMENTABLE)

## PURPOSE

Define an AST-based semantic merge system that is directly
implementable.

------------------------------------------------------------------------

# 1. CORE MODEL

We operate on:

-   BASE (original file)
-   LOCAL (current codebase)
-   NEW (LLM output)

Goal: → merge NEW into LOCAL using BASE as reference

------------------------------------------------------------------------

# 2. PARSING

Use Python AST:

``` python
import ast

tree = ast.parse(code)
```

Extract:

-   functions
-   classes
-   imports

------------------------------------------------------------------------

# 3. NODE IDENTIFICATION

Each node gets:

    (type, name, start_line, end_line)

Example:

    ("function", "calculate_age", 10, 25)

------------------------------------------------------------------------

# 4. SPAN EXTRACTION

Use AST + tokenize:

``` python
import tokenize
```

Map node → exact source span

------------------------------------------------------------------------

# 5. DIFF STRATEGY (FUNCTION LEVEL)

Compare:

BASE vs LOCAL BASE vs NEW

Cases:

1.  unchanged → skip
2.  changed in NEW only → apply
3.  changed in LOCAL only → keep
4.  changed in both → conflict

------------------------------------------------------------------------

# 6. MERGE RULES

## SAFE MERGE

If function exists only in NEW: → append

If function unchanged in LOCAL: → replace with NEW

------------------------------------------------------------------------

## CONFLICT

If both changed:

    LOCAL != BASE
    NEW != BASE

→ conflict

------------------------------------------------------------------------

# 7. CONFLICT RESOLUTION LOOP

Send to LLM:

INPUT:

-   BASE version
-   LOCAL version
-   NEW version

OUTPUT FORMAT:

\[RESOLVED\]

``` python
<merged function>
```

Validate → replace

------------------------------------------------------------------------

# 8. IMPORT MERGE

-   Parse imports via AST
-   Union of LOCAL + NEW
-   Remove duplicates

------------------------------------------------------------------------

# 9. FILE REBUILD

Reconstruct:

1.  imports
2.  functions/classes
3.  remaining code

------------------------------------------------------------------------

# 10. VALIDATION

Before commit:

-   ast.parse() must pass
-   no missing imports
-   no syntax error

------------------------------------------------------------------------

# 11. PIPELINE

Parse → Extract Nodes → Diff → Merge → Validate → Commit

------------------------------------------------------------------------

# 12. FAILURE CONDITIONS

-   AST parse fail
-   unresolved conflict
-   invalid syntax

→ retry or abort

------------------------------------------------------------------------

# RESULT

-   Function-level precision
-   Minimal diff
-   Safe merge
-   Deterministic behavior
