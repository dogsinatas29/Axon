## 🏛️ Sovereign Protocol - Master Hub Specification (v0.0.22+)
**"모든 구현은 규약을 따르며, 모든 통합은 마스터 허브에서 증명된다."**

### 1. 단일 진실 공급원 (SSOT) 체계
- **[architecture.md](file:///home/dogsinatas/rust_project/axon/architecture.md)**: 전체 시스템의 권한 지도 및 계층 구조를 총괄하는 최상위 허브.
- **모듈화 스펙**:
  - `spec`: 최상위 요구사항 및 시스템 정의.
  - `schema (SSOT)`: 데이터 및 행위의 엄격한 제약 사항 정의.
  - `validator`: 스키마 준수 여부를 검증하는 로직.
  - `repair_prompt`: 검증 실패 시 모델에게 전달할 교정 가이드.

### 2. 팀 기반 독재적 효율성 (Sovereign Authority)
- **Deterministic Load Order**: 시스템 문서는 반드시 [spec -> schema -> validator -> repair prompt -> loop] 순서로 로드되어야 함.
- **Fixed-point Convergence**: 루프는 최종적으로 IR(Intermediate Representation)이 변하지 않는 고정점에 도달해야 함.

## 🧠 상세 설계 및 로직
- **Load Ordering**: 비결정론적 파일 읽기를 방지하기 위해 명시적인 시퀀셜 로더 구현.
- **SSOT Binding**: 모든 생성 로직은 사전에 로드된 스키마 IR을 참조하여 컨텍스트를 제한.
- **Convergence Logic**: 에이전트의 출력이 이전 상태와 동일해질 때까지(또는 스키마를 완벽히 충족할 때까지) 반복 수정 수행.

## 🛠️ 기술적 변경 사항
- **Daemon Update:** `axon-daemon`에 `BootstrapManager` 추가.
- **Validator Update:** Rust 기반의 스키마 유효성 검사기 강화.
- **Prompt Update:** `repair_prompt` 템플릿 최적화.

## ⚠️ 예외 처리 및 주의 사항
- 루프가 발산(Divergence)하거나 무한 루프에 빠질 가능성 대비 (Max retry limit 필수).
- 스키마 자체가 모순될 경우의 에러 핸들링.
