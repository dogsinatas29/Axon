# v0.0.22 End-to-End Deterministic LLM Pipeline (Rust)

## 1. Overview

This document provides a minimal, executable Rust pipeline that
connects:

-   LLM (generation + repair)
-   Validator (deterministic checks)
-   Repair prompt (controlled fixes)
-   Convergence loop (fixed-point)

## 4. Validator

``` rust
#[derive(Debug, Clone, Serialize)]
pub struct ValidationError {
    pub path: String,
    pub kind: String,
}

pub fn validate(ir: &IR) -> Vec<ValidationError> {
    let mut errors = vec![];

    for (i, file) in ir.files.iter().enumerate() {
        if file.name.trim().is_empty() {
            errors.push(err(format!("files[{}].name", i), "EmptyField"));
        }

        for (j, func) in file.functions.iter().enumerate() {
            if func.name.trim().is_empty() {
                errors.push(err(format!("files[{}].functions[{}].name", i, j), "EmptyField"));
            }
        }
    }

    errors
}

fn err(path: String, kind: &str) -> ValidationError {
    ValidationError {
        path,
        kind: kind.to_string(),
    }
}
```

## 5. LLM Call

``` rust
async fn call_llm(prompt: &str) -> anyhow::Result<String> {
    let client = reqwest::Client::new();

    let res = client
        .post("http://localhost:8000/v1/chat/completions")
        .json(&serde_json::json!({
            "model": "gpt-4o-mini",
            "messages": [
                {"role": "user", "content": prompt}
            ],
            "temperature": 0
        }))
        .send()
        .await?;

    let json: serde_json::Value = res.json().await?;
    let content = json["choices"][0]["message"]["content"]
        .as_str()
        .unwrap_or("")
        .to_string();

    Ok(content)
}
```

## 6. Repair Prompt Builder

``` rust
fn build_repair_prompt(ir: &IR, errors: &[ValidationError]) -> String {
    format!(
r#"
You are a strict IR repair engine.

Rules:
- Fix ONLY fields in error list
- DO NOT modify valid fields
- DO NOT add keys
- Output valid JSON ONLY

Input:
{input}

Errors:
{errors}

Return fixed JSON only.
"#,
        input = serde_json::to_string_pretty(ir).unwrap(),
        errors = serde_json::to_string_pretty(errors).unwrap()
    )
}
```

## 7. Initial Fill

``` rust
async fn initial_fill(spec: &str) -> anyhow::Result<IR> {
    let prompt = format!(
        r#"
Generate JSON IR matching schema.

Schema:
{{
  "files": [
    {{
      "name": "",
      "responsibility": "",
      "functions": [
        {{
          "name": "",
          "args": [],
          "returns": ""
        }}
      ]
    }}
  ]
}}

Spec:
{}

Return JSON only.
"#,
        spec
    );

    let raw = call_llm(&prompt).await?;
    let ir: IR = serde_json::from_str(&raw)?;
    Ok(ir)
}
```

## 8. Convergence Loop

``` rust
pub async fn run_pipeline(spec: &str) -> anyhow::Result<IR> {
    let mut ir = initial_fill(spec).await?;
    let mut prev_hash = String::new();

    for _ in 0..10 {
        let errors = validate(&ir);

        if errors.is_empty() {
            println!("Converged");
            return Ok(ir);
        }

        let prompt = build_repair_prompt(&ir, &errors);
        let raw = call_llm(&prompt).await?;

        let new_ir: IR = match serde_json::from_str(&raw) {
            Ok(v) => v,
            Err(_) => continue,
        };

        let hash = format!("{:?}", new_ir);

        if hash == prev_hash {
            break;
        }

        prev_hash = hash;
        ir = new_ir;
    }

    Err(anyhow::anyhow!("Failed to converge"))
}
```

## 9. Main Entry

``` rust
#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let spec = "simple database module with query execution";

    let result = run_pipeline(spec).await?;

    println!("{}", serde_json::to_string_pretty(&result)?);

    Ok(())
}
```

## 10. Summary

This pipeline enforces:

-   Schema-driven structure
-   Deterministic validation
-   Localized repair
-   Fixed-point convergence

LLM is constrained into a repair loop, not free generation.
