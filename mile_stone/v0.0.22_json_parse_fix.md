# v0.0.22 LLM JSON Parse Error Handling Guide

## 1. Error Description

    EOF while parsing a value at line 1 column 0

### Meaning

-   Empty string ("")
-   Whitespace only
-   Non-JSON response

------------------------------------------------------------------------

## 2. Root Cause

The failure occurs at:

``` rust
let ir: IR = serde_json::from_str(&raw)?;
```

This means `raw` is not valid JSON.

------------------------------------------------------------------------

## 3. Common Causes

### (1) Empty LLM Response

-   Model returned no content
-   API response malformed

### (2) Extra Text Around JSON

Example:

    Sure! Here is your JSON:

    { ... }

### (3) HTTP Failure

-   500 / timeout / empty body

------------------------------------------------------------------------

## 4. Required Debugging

``` rust
println!("RAW LLM OUTPUT:\n{}", raw);
```

------------------------------------------------------------------------

## 5. JSON Extraction Fix

``` rust
fn extract_json(raw: &str) -> Option<String> {
    let start = raw.find('{')?;
    let end = raw.rfind('}')?;
    Some(raw[start..=end].to_string())
}
```

------------------------------------------------------------------------

## 6. Safe Parsing

``` rust
let cleaned = extract_json(&raw)
    .ok_or_else(|| anyhow::anyhow!("No JSON found"))?;

let ir: IR = serde_json::from_str(&cleaned)?;
```

------------------------------------------------------------------------

## 7. Hardened LLM Call

``` rust
async fn call_llm(prompt: &str) -> anyhow::Result<String> {
    let client = reqwest::Client::new();

    let res = client
        .post("http://localhost:8000/v1/chat/completions")
        .json(&serde_json::json!({
            "model": "gpt-4o-mini",
            "messages": [
                {"role": "user", "content": prompt}
            ],
            "temperature": 0
        }))
        .send()
        .await?;

    if !res.status().is_success() {
        return Err(anyhow::anyhow!("HTTP error: {}", res.status()));
    }

    let json: serde_json::Value = res.json().await?;

    let content = json
        .get("choices")
        .and_then(|c| c.get(0))
        .and_then(|c| c.get("message"))
        .and_then(|m| m.get("content"))
        .and_then(|c| c.as_str())
        .ok_or_else(|| anyhow::anyhow!("Invalid LLM response"))?;

    Ok(content.to_string())
}
```

------------------------------------------------------------------------

## 8. Prompt Constraint (Critical)

Always include:

    Return ONLY valid JSON.
    Do NOT include explanations.
    Do NOT include markdown.
    Do NOT include backticks.

------------------------------------------------------------------------

## 10. Conclusion

This error is not a Rust issue.

It is caused by invalid or empty LLM output.

Solution: - Enforce strict JSON output - Extract JSON safely - Validate
before parsing
