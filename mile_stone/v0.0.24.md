## 🧠 상세 설계 및 로직
- **Pollution Shield**: `strip_markdown` 유틸리티를 통한 소스 코드 내 마크다운 완전 배제 및 `axon-agent` 레벨의 자동 정제 통합.
- **Scope Control**: `target_file` 외의 파일 수정 시도를 `SCOPE_VIOLATION`으로 즉각 기각하며, `detect_scope_violation` 패턴 검증 도입.
- **Execution Optimization**: 프로젝트의 마지막 태스크에서만 무거운 실행 하네스를 돌리도록 최적화하여 공정 속도 개선.
- **Senior Gate Hardening**: `[APPROVE]`, `[COMPLIANT]` 등 대괄호 기반의 엄격한 태그 매칭 도입으로 승인 오판 방지.
- **UI Language Sync**: 데몬의 `locale` 설정을 Studio UI와 동기화하여 `i18n` 사전 기반의 다국어 인터페이스 구현.

## 📦 Release Notes (v0.0.24)

### [KR]
- **무결성 강화**: Newline-agnostic 비교 로직 도입으로 허위 Target Mismatch 제거.
- **하네스 완화**: 최소 코드 크기 제한 60B로 조정 (단순 핸들러 지원).
- **감사 강화**: 모든 실패 시도 DB 저장 및 연결성 가드(Connectivity Guard) 도입.

### [EN]
- **Integrity**: Newline-agnostic comparison to prevent false-positive Target Mismatch.
- **Threshold**: Lowered minimum code size to 60B for short handlers.
- **Audit**: Forced DB logging for all proposals and added Connectivity Guard.
