# SYNAPSE v0.0.24 Diff Enforcement (Phase 4)

## Goal

Ensure all modifications are strictly limited to a single target file
using diff format.

------------------------------------------------------------------------

## Concept

LLM output must follow:

``` diff
--- target.rs
+++ target.rs
@@
+ fn example() {}
```

------------------------------------------------------------------------

## Validation Rules

1.  Output must start with target file diff header
2.  No additional file headers allowed
3.  Only one file allowed per diff

------------------------------------------------------------------------

## Implementation

``` python
def is_single_file_diff(diff_text: str, target: str):
    lines = diff_text.strip().splitlines()

    if not lines[0].startswith(f"--- {target}"):
        return False

    file_headers = [l for l in lines if l.startswith("--- ")]
    if len(file_headers) != 1:
        return False

    return True
```

------------------------------------------------------------------------

## Integration

``` python
if not is_single_file_diff(output, target_file):
    return False
```

------------------------------------------------------------------------

## Optional Strict Mode

``` python
def reject_multi_file(diff_text):
    return diff_text.count("--- ") > 1
```

------------------------------------------------------------------------

## Test Harness

``` python
def test_valid_diff():
    diff = """--- a.rs
+++ a.rs
@@
+ fn test() {}"""
    assert is_single_file_diff(diff, "a.rs") == True


def test_invalid_diff():
    diff = """--- a.rs
--- b.rs"""
    assert is_single_file_diff(diff, "a.rs") == False
```

------------------------------------------------------------------------

## Result

-   Multi-file edits impossible
-   Structural control over changes
-   Enables safe future parallelization
