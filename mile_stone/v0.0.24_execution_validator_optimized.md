# SYNAPSE v0.0.24 Execution Validator (Optimized)

## Goal

Ensure generated code is compilable and executable, while avoiding
unnecessary failures during partial generation.

------------------------------------------------------------------------

## Core Idea

Execution Validator should run **ONLY at the final stage** of file
generation.

------------------------------------------------------------------------

## Pipeline (Optimized)

LLM → Pre-clean → Validator → Write → (FINAL NODE) → Execution Validator

------------------------------------------------------------------------

## Why Optimization is Needed

### Problem (Before)

Execution Validator runs on every file:

-   Early files fail (incomplete project)
-   False negatives
-   Retry explosion

------------------------------------------------------------------------

### Solution (After)

Run Execution Validator ONLY when:

-   All files are generated
-   Project is complete

------------------------------------------------------------------------

## Implementation Strategy

### Step 1 --- Detect Final Node

``` python
def is_final_node(step_index, total_steps):
    return step_index == total_steps - 1
```

------------------------------------------------------------------------

### Step 2 --- Modify Executor Loop

``` diff
for i, step in enumerate(plan):
```

## Execution Validator (Same as v0.0.24)

### Compile Check

``` python
def compile_check(project_path: str):
    import subprocess
    result = subprocess.run(
        ["cargo", "check"],
        cwd=project_path,
        capture_output=True,
        text=True
    )
    return result.returncode == 0, result.stderr
```

------------------------------------------------------------------------

### Run Check

``` python
def run_check(project_path: str):
    import subprocess
    result = subprocess.run(
        ["python", "main.py"],
        cwd=project_path,
        capture_output=True,
        text=True,
        timeout=5
    )
    return result.returncode == 0, result.stdout, result.stderr
```

------------------------------------------------------------------------

### Integration

``` python
def execution_validate(project_path: str):
    ok, err = compile_check(project_path)
    if not ok:
        return False, f"COMPILE_FAIL: {err}"

    ok, out, err = run_check(project_path)
    if not ok:
        return False, f"RUNTIME_FAIL: {err}"

    return True, "OK"
```

------------------------------------------------------------------------

## Success Criteria

-   No false failure during partial generation
-   Compile success at final stage
-   Runtime success at final stage

------------------------------------------------------------------------

## Key Rule

Execution validation MUST NOT run before the system is complete.

------------------------------------------------------------------------

## Summary

This optimization ensures:

-   Stable pipeline execution
-   Accurate validation timing
-   Reduced retry noise

------------------------------------------------------------------------

## One-Line Principle

Validate only when the system is whole.
