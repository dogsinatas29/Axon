# SYNAPSE v0.0.24 Scope Control (Basic)

## Goal

Prevent unintended cross-file modification by LLM with minimal
implementation cost.

------------------------------------------------------------------------

## Strategy

Introduce Scope Control in incremental phases:

Detect → Limit → Enforce → Isolate

------------------------------------------------------------------------

## Phase 1: Output Filtering (Detection)

### Goal

Detect and reject cross-file modification attempts.

### Implementation

``` python
def detect_scope_violation(content: str):
    forbidden_patterns = [
        "other_file.rs",
        "mod ",
        "use crate::",
        "../",
    ]
    return any(p in content for p in forbidden_patterns)
```

Integrate into validator:

``` python
def validate(content: str):
    if "```" in content:
        return False

    if detect_scope_violation(content):
        return False

    return True
```

------------------------------------------------------------------------

## Phase 2: Target File Binding

### Goal

Restrict LLM modification scope to a single file.

### Implementation

Execution context:

``` python
task = {
    "target_file": "calculate.rs"
}
```

LLM instruction:

-   Modify ONLY the target file
-   Do NOT reference other files

------------------------------------------------------------------------

## Phase 3: Write Gate

### Goal

Physically prevent writing to non-target files.

### Implementation

``` python
def write_file(target_path, content):
    with open(target_path, "w") as f:
        f.write(content)
```

------------------------------------------------------------------------

## Phase 4: Diff Enforcement (Optional)

### Goal

Ensure structured and limited modification.

``` python
def is_single_file_diff(diff_text, target):
    return diff_text.startswith(f"--- {target}")
```

------------------------------------------------------------------------

## Phase 5: File System Sandbox (Advanced)

### Goal

Full isolation.

Structure:

/tmp/sandbox/ target.rs (write) others.rs (read-only)

------------------------------------------------------------------------

## Recommended Initial Setup

Apply:

-   Phase 1 (Detection)
-   Phase 3 (Write Gate)

------------------------------------------------------------------------

## Success Criteria

-   Cross-file modification attempts are rejected
-   Only target file is modified
-   No unintended side effects

------------------------------------------------------------------------

## Summary

Scope Control is not implemented in one step.

It is progressively enforced:

Detection → Restriction → Enforcement → Isolation
