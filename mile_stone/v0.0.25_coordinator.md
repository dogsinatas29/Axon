# 🧠 AXON v0.0.25 Coordinator Specification

## 0. Purpose

Introduce a Coordinator Layer to control:
- Execution order
- Worker assignment
- File-level isolation
- State transitions

---

## 1. Core Concept

Coordinator = Planner
Worker = Executor

---

## 2. Responsibilities

- DAG execution control
- Task scheduling
- File locking
- Scope enforcement
- State machine control

---

## 3. State Machine

PENDING → READY → RUNNING → VALIDATING → COMMITTED / FAILED

---

## 4. Minimal Structures (Rust)

struct Task {
    id: String,
    target_file: String,
    dependencies: Vec<String>,
    status: TaskStatus,
}

enum TaskStatus {
    Pending,
    Ready,
    Running,
    Validating,
    Committed,
    Failed,
}

---

## 5. Core Rules

1. Dependency satisfied before execution
2. One file = one worker
3. Only target_file can be modified
4. Deterministic execution order

---

## 6. Execution Logic

- Check dependencies
- Check file lock
- Dispatch task
- Lock file
- Execute
- Unlock on completion

---

## 7. Phase Strategy

Phase 1: Single worker + Coordinator  
Phase 2: Multi-worker with locks  

---

## 8. Success Criteria

- No stub cascade
- No race condition
- Deterministic pipeline
