# v0.0.25_execution_final_gate.md

## Execution Validator 포함 최종 Gate (compile + run + reject)

---

# 0. Goal

**“실행 가능한 코드만 통과시키는 최종 방어선 구축”**

다음 조건을 모두 만족해야만 승인된다:

* 컴파일 성공
* 실행 성공
* 런타임 오류 없음
* 검증 규칙 통과
* 시니어 승인 명시 태그 존재

---

# 1. Final Gate 전체 흐름

```
[Coordinator Lock]
        ↓
[Single-file Sandbox]
        ↓
[Static Validator]
        ↓
[Execution Validator]
    ├─ Compile Check
    └─ Run Check
        ↓
[Senior Gate (STRICT)]
        ↓
[Atomic Commit]
```

---

# 2. Execution Validator 설계

## 2.1 목표

기존 문제:

* “형식만 맞는 코드” 통과
* “컴파일 안 되는 코드” 통과
* “실행 시 터지는 코드” 통과

해결:
→ **실제 컴파일 + 실행 검증**

---

## 2.2 Compile Check

```rust
use std::process::Command;

fn compile_check() -> Result<(), String> {
    let output = Command::new("cargo")
        .arg("build")
        .output()
        .map_err(|e| e.to_string())?;

    if !output.status.success() {
        return Err(format!(
            "F_COMPILE_FAIL\n{}",
            String::from_utf8_lossy(&output.stderr)
        ));
    }

    Ok(())
}
```

---

## 2.3 Run Check

```rust
fn run_check() -> Result<(), String> {
    let output = Command::new("cargo")
        .arg("run")
        .output()
        .map_err(|e| e.to_string())?;

    if !output.status.success() {
        return Err(format!(
            "F_RUNTIME_FAIL\n{}",
            String::from_utf8_lossy(&output.stderr)
        ));
    }

    Ok(())
}
```

---

## 2.4 Execution Validator 통합

```rust
fn execution_validate() -> Result<(), String> {
    compile_check()?;
    run_check()?;
    Ok(())
}
```

# 4. Senior Gate (Strict Mode)

## 기존 문제

```
contains("APPROVE") → 오탐 발생
```

---

## 해결

```rust
fn senior_gate(content: &str) -> Result<(), String> {
    if content.contains("[REJECT]") {
        return Err("F_REJECTED_BY_SENIOR".into());
    }

    if !content.contains("[APPROVE]") {
        return Err("F_NO_APPROVAL_TAG".into());
    }

    Ok(())
}
```

# 5. Atomic Commit (Rollback 보장)

## 문제

* 실패 시 파일 오염 발생

---

## 해결 구조

```rust
fn atomic_commit(target: &str, new_content: &str) -> Result<(), String> {
    let backup = std::fs::read_to_string(target).unwrap_or_default();
    let temp = format!("{}.tmp", target);

    std::fs::write(&temp, new_content).map_err(|e| e.to_string())?;

    if let Err(e) = execution_validate() {
        std::fs::write(target, backup).ok();
        return Err(format!("F_ROLLBACK\n{}", e));
    }

    std::fs::rename(temp, target).map_err(|e| e.to_string())?;

    Ok(())
}
```

# 7. 실패 코드 정의

| 코드                   | 의미          |
| -------------------- | ----------- |
| F_STUB               | Stub 코드     |
| F_MARKDOWN           | Markdown 오염 |
| F_HARDCODE           | 하드코딩        |
| F_COMPILE_FAIL       | 컴파일 실패      |
| F_RUNTIME_FAIL       | 실행 실패       |
| F_REJECTED_BY_SENIOR | 시니어 거절      |
| F_NO_APPROVAL_TAG    | 승인 태그 없음    |
| F_ROLLBACK           | 롤백 발생       |

# 8. 테스트 시나리오

## Case 1: Stub

→ FAIL (F_STUB)

## Case 2: Markdown 포함

→ FAIL (F_MARKDOWN)

## Case 3: 컴파일 실패

→ FAIL (F_COMPILE_FAIL)

## Case 4: 런타임 에러

→ FAIL (F_RUNTIME_FAIL)

## Case 5: 승인 태그 없음

→ FAIL (F_NO_APPROVAL_TAG)

## Case 6: 정상 코드 + [APPROVE]

→ PASS
