# 📄 v0.0.25_final_gate_execution_validator.md

## 🎯 Goal

Execution Validator + Senior JSON Gate를 결합하여:

👉 **컴파일 + 실행 + 구조 검증 + 논리 검증 + 승인 판단까지 하나의 최종 게이트에서 처리**

즉, 이 단계 통과 = **배포 가능한 코드**

---

## 🧱 Final Gate 구조 (단일 파이프라인)

```
[PHASE 0] Dry Run (No Disk Write)
    ↓
[PHASE 1] Signature Validation
    ↓
[PHASE 2] Execution Validator (compile + run)
    ↓
[PHASE 3] Senior JSON Review
    ↓
[PHASE 4] Physical Commit (Atomic)
```

---

## 🔍 PHASE 0 — Dry Run (필수)

### 목적

* destructive rewrite 차단
* 스텁 차단
* 디스크 오염 방지

### 규칙

* 기존 코드 + patch → in-memory merge
* 실패 시 **디스크 write 금지**

---

## 🔍 PHASE 1 — Signature Validation

### 검증 항목

* required function 존재 여부
* 함수 시그니처 일치 여부

### 실패 시

```
❌ REJECT (즉시 중단)
```

---

## 🔍 PHASE 2 — Execution Validator (핵심)

### 목적

👉 "돌아가는 코드만 통과"

---

### Rust 기준 실행

```bash
cargo check
cargo run
```

---

### Python 기준 실행

```bash
python main.py
```

---

### 검증 조건

| 항목        | 조건                   |
| --------- | -------------------- |
| 컴파일       | 성공                   |
| 런타임       | panic / exception 없음 |
| exit code | 0                    |

---

### Rust 예시 코드

```rust
fn execution_validate(project_path: &str) -> Result<()> {
    let status = Command::new("cargo")
        .arg("check")
        .current_dir(project_path)
        .status()?;

    if !status.success() {
        return Err(anyhow!("COMPILE_FAILED"));
    }

    let run = Command::new("cargo")
        .arg("run")
        .current_dir(project_path)
        .status()?;

    if !run.success() {
        return Err(anyhow!("RUNTIME_FAILED"));
    }

    Ok(())
}
```

---

### 실패 시

```
❌ REJECT (즉시 중단)
```

## 🔍 PHASE 3 — Senior JSON Gate

### 입력

* target_file
* expected_functions
* existing_code
* merged_code

---

### 출력 (강제 JSON)

```json
{
  "decision": "APPROVE",
  "reason": "...",
  "violations": []
}
```

---

### 처리 로직

```rust
let review: ReviewResult = serde_json::from_str(&response)
    .map_err(|_| anyhow!("INVALID_JSON"))?;

match review.decision.as_str() {
    "APPROVE" => proceed(),
    "REJECT" => abort(review.violations),
    _ => abort_invalid()
}
```

---

### Fail-Closed

| 상황               | 처리     |
| ---------------- | ------ |
| JSON 파싱 실패       | REJECT |
| decision 없음      | REJECT |
| invalid decision | REJECT |

---

## 🔍 PHASE 4 — Physical Commit (Atomic)

### 조건

👉 모든 Phase 통과 시에만 실행

---

### 방식

```rust
write_temp_file()
fsync()
atomic_rename()
```

---

### 보장

* 중간 실패 시 파일 변경 없음
* rollback 필요 없음 (이미 Dry Run)

| 문제              | 해결              |
| --------------- | --------------- |

| partial rewrite | Signature 단계 차단 |
