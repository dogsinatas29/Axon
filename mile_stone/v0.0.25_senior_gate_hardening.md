# v0.0.25_senior_gate_hardening.md

## 🎯 Goal

Senior Gate의 문자열 기반 승인 판정 취약점을 제거하고, **결정(Decision)을 구조적으로 강제하여 오판 승인(false positive)**을 원천 차단한다.

---

## 🔥 Problem Definition

### 기존 구조 (치명적 결함)

```rust
if content.contains("APPROVE") {
    pass
}
```

### 문제점

* "REJECT ... APPROVE" → 통과됨 ❌
* 문맥 기반 오판 발생
* LLM 출력 특성상 신뢰 불가
* 최종 Gate 무력화 가능

---

## ✅ Design Principles

1. **명시적 결정만 허용**
2. **모호성 = 실패**
3. **문자열 탐색 금지**
4. **결정은 구조화된 형태로 파싱**

---

## 🧩 Phase A — Tag-based Hardening (즉시 적용)

### 1️⃣ Decision Enum

```rust
enum ReviewDecision {
    Approve,
    Reject,
}
```

---

### 2️⃣ Strict Parser

```rust
fn parse_decision(content: &str) -> Option<ReviewDecision> {
    let has_approve = content.contains("[APPROVE]");
    let has_reject = content.contains("[REJECT]");

    match (has_approve, has_reject) {
        (true, false) => Some(ReviewDecision::Approve),
        (false, true) => Some(ReviewDecision::Reject),
        _ => None, // 🔴 ambiguous or missing
    }
}
```

---

### 3️⃣ Gate Enforcement

```rust
match parse_decision(content) {
    Some(ReviewDecision::Approve) => {
        // proceed
    }
    Some(ReviewDecision::Reject) => {
        return Err("Rejected by Senior");
    }
    None => {
        return Err("Invalid or ambiguous decision");
    }
}
```

---

## ⚠️ Critical Rule

```text
NO TAG → FAIL
BOTH TAGS → FAIL
```

## 🚀 Phase B — JSON Structured Decision (Next Step)

### LLM Output Format

```json
{
  "decision": "APPROVE",
  "reason": "All validations passed"
}
```

---

### Rust Struct

```rust
use serde::Deserialize;

#[derive(Deserialize)]
struct ReviewResult {
    decision: String,
    reason: String,
}
```

---

### Strict Handling

```rust
fn handle_review(result: ReviewResult) -> Result<(), String> {
    match result.decision.as_str() {
        "APPROVE" => Ok(()),
        "REJECT" => Err("Rejected by Senior".into()),
        _ => Err("Invalid decision".into()),
    }
}
```
