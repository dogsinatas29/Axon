# 📄 v0.0.25_senior_prompt_structured.md

## 🎯 Goal

Senior 판단을 **완전 구조화(JSON only)**로 강제하여:

* 문자열 기반 판정 제거 (`contains("APPROVE")` 제거)
* 자연어 모호성 제거
* Fail-closed 구조 확립 (파싱 실패 = 즉시 REJECT)
* 파이프라인 결정의 완전한 결정론 확보

## 🔧 Daemon Side Parsing (필수 변경)

### ❌ 기존 (취약)

```rust
if content.contains("APPROVE") {
    approve();
}
```

---

### ✅ 변경 (Strict JSON Parsing)

```rust
#[derive(serde::Deserialize)]
struct ReviewResult {
    decision: String,
    reason: String,
    violations: Vec<Violation>,
}

#[derive(serde::Deserialize)]
struct Violation {
    #[serde(rename = "type")]
    vtype: String,
    message: String,
}

fn handle_review(content: &str) -> Result<()> {
    let parsed: ReviewResult = serde_json::from_str(content)
        .map_err(|_| anyhow::anyhow!("INVALID_JSON_FROM_SENIOR"))?;

    match parsed.decision.as_str() {
        "APPROVE" => approve(),
        "REJECT" => reject(parsed.violations),
        _ => Err(anyhow::anyhow!("INVALID_DECISION"))
    }
}
```

## 🔒 Fail-Closed 정책

| 상황            | 처리     |
| ------------- | ------ |
| JSON 파싱 실패    | REJECT |
| decision 없음   | REJECT |
| decision 이상 값 | REJECT |
| extra text 포함 | REJECT |
