## 🧠 상세 설계 및 로직

### 1. 결정론적 상태 머신 파이프라인 (Phase 1.4)
- **Stage**: `Skeleton` -> `HeaderGen` -> `ImplGen` -> `Build` -> `Runtime` -> `Sync` -> `Complete`
- **Logic**: 각 단계가 물리적으로 고립되어 있으며, 성공 시에만 다음 단계로 승격(Promotion)되는 엄격한 상태 제어 루프.

### 2. 고도화된 실패 분류기 및 진단 (Phase 1.4.5)
- **Normalizer**: 로그 노이즈 제거 및 소문자화.
- **Precision Matcher**: `expected value`, `eof`, `trailing characters` 등 LLM 파싱 에러를 `JsonError`로 정밀 타격.
- **Diagnostic Transparency**: 로그 출력 시 실제 에러 메시지를 노출(`error="..."`)하여 관리자의 가시성 확보.

### 3. 에스컬레이션 사다리 및 복구 (Phase 1.4.7)
- **Recovery Strategy**: `Patch` -> `RegenerateFile` -> `RetryStage` -> `FullReset`
- **Header Protection**: 헤더 파일 수정 시 재생성 대신 공정 재점검을 강제하여 아키텍처 정합성 보호.

### 4. 상태 기반 프롬프트 엔진 (Phase 1.4.6)
- **PromptBuilder**: `[ROLE]`, `[TASK]`, `[CONSTRAINTS]`, `[INPUT]`의 4단 구조 강제.
- **Dynamic Injection**: 실패 원인에 따른 '처방전(Hint)'을 프롬프트에 자동 주입.
