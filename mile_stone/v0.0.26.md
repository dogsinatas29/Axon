### 2. 결정론적 출력 채널 (The Iron Pipeline - 3중 고정)
- **Layer 1: Sampling Stabilization**: 전 드라이버의 `temperature`를 0.0으로 고정하고, `<JSON_END>`를 Stop Sequence로 설정하여 무작위성 제거.
- **Layer 2: Output Boundary Enforcement**: `<JSON_START>`와 `<JSON_END>` 토큰 사이에 JSON을 감싸는 **Hard Envelope** 규격 강제.
- **Layer 3: 2-Phase Self-Healing**: 1차 파싱 실패 시, 오류 메시지와 함께 원본을 `repair_ir_pass` 요원에게 전달하여 즉각적인 자가 수복 수행.

### 4. 7B 모델 최적화 (7B Optimization - NEW)
- **Prompt Diet**: 7B 모델의 인지 부하 감소를 위해 지시 사항을 핵심 불렛 포인트 위주로 압축.
- **Raw Logging**: 추출 실패 시 LLM의 원본 응답을 로그에 기록하여 비결정적 실패 원인 분석 가시성 확보.
- **Diagnostic Expansion**: `failed to find`, `envelope` 등 하드닝 전용 에러 패턴을 진단기에 통합.
- **7B Resilient Parser**: JSON 내부를 재귀적으로 탐색하여 `components` 루트를 찾아내고, 파일 확장자를 자동 추론하는 지능형 파서 탑재.
- **C Modularity Enforcement**: C 프로젝트 설계 시 `.h`와 `.c` 파일의 엄격한 분리를 강제하는 검증 게이트 및 프롬프트 규칙 도입.
- **HeaderGen Hardening**: 태스크 추출 공정에도 Hard Envelope 및 자가 수복 로직을 이식하고, 파싱 실패 시 단계별 리트라이(Stage Retry)를 수행하도록 개선.

## 🧠 상세 설계 및 로직

### 1. Hard Envelope 추출기 (`extract_enveloped_json`)
- 모델이 쏟아내는 잡설 속에서 `<JSON_START>`와 `<JSON_END>` 사이의 데이터만 정밀하게 타격하여 추출.
- 실패 시 Curly Braces(`{}`) 기반의 Fallback 로직 가동.

### 2. 자가 수복 공정 (`Repair Pass`)
- 파싱 에러 발생 시, 에이전트에게 **[ERROR_MSG]**와 **[BROKEN_JSON]**을 다시 던져 "Fix ONLY JSON syntax" 지시 하달.
- 재시도 횟수를 소모하기 전, 1회의 즉각적인 수리 기회를 부여하여 토큰 및 시간 효율 극대화.

### 3. 드라이버 하드닝 (Model Driver Hardening)
- Gemini: `generationConfig` 주입 (Temperature=0, StopSequences).
- Claude/OpenAI: `temperature: 0.0`, `stop_sequences` 필드 적용.
- Ollama: `options.temperature: 0.0`, `options.stop` 고정.

## ⚠️ 예외 처리 및 주의 사항
- **Stop Sequence Conflict**: 모델이 `<JSON_END>`를 만나면 즉시 중단되므로, 불필요한 뒷부분 텍스트 유입이 원천 차단됨.
- **Empty Response**: 빈 응답 수신 시 즉각적인 재요청 시퀀스 발동.
