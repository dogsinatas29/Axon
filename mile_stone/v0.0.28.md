## 🧠 상세 설계 및 로직

### 1. 웹 뷰어 제어 기능 (The Control Panel)
- **Lock-in**: 확정된 코드를 성역화하고 Architecture.md에 [✅ Locked] 마크업 수행.
- **Pause / Resume (작업 일시 정지/재개)**:
  - 특정 스레드(태스크)를 일시 정지시키면, 해당 에이전트에게 전송되는 AXP 패킷에 HOLD 플래그를 실어 보냄.
  - 에이전트는 '대기' 상태로 전환되어 토큰 소모를 중단하고, 사용자가 Resume을 누를 때까지 연산을 멈춤.
- **Create Thread (스레드 즉시 개설)**:
  - 사용자가 웹에서 새 주제를 입력하면, 최상위 노드(Root)가 이를 가로채서 Architecture.md 하단에 적절한 헤더를 자동으로 추가함.
  - 파일이 변하면 데몬의 Watcher가 이를 감지해 병렬 작업 라인을 즉시 가동함.

### 2. 프로젝트 격리 및 별도 뷰어 (Multi-Project View)
- **Namespace Isolation**: 각 프로젝트는 고유의 ID(또는 폴더 경로)를 가지며, 데몬 내에서 별도의 컨텍스트로 관리됨.
- **Independent View**: `localhost:8080/project-a`, `localhost:8080/project-b` 식으로 엔드포인트를 분리함.
- 한 화면에서 여러 프로젝트를 탭(Tab) 형태로 전환하거나, 별도의 브라우저 창에서 독립적으로 모니터링함.

### 🏭 3. AXON: Automated Software Factory (Scenario)
- **Line Setup**: Boss(User)가 설계도를 던지면, Senior(Claude)가 공정을 감시하고 Junior(Gemini)들이 병렬로 태스크를 수행하는 구조.
- **Production Loop**: 주니어가 코드를 제출하면 시니어가 검토(Review)하고 승인 시 다음 태스크로 자동 이동.
- **Final Build**: 모든 하위 태스크 완료 시 시니어가 최종 합본을 출력하고 사용자가 QA 수행.

### 🛠️ 4. Bug Reporting & Debugging Workflow
- **Multi-modal Input**: 스크린샷(UI 버그) 및 텍스트 로그(런타임 에러)를 웹 뷰어에 직접 투입.
- **Debugging Signal**: 에러 리포트 발생 시 즉시 관련 스레드에 인터럽트(Interrupt)를 송신하여 수정 작업 급파.
- **Human-in-the-Loop QA**: 사용자가 직접 실행 후 에러 메시지를 박아줌으로써 AI의 추론 비용(토큰)을 획기적으로 절감.

### 🥊 5. Adversarial Mode (The Colosseum of LLMs)
- **Adversarial Persona**: 시니어와 주니어가 게시판에서 서로의 논리적 허점을 공격하며 코드를 최적화.
- **Debate-First Mode**: 상호 비판 3회 전까지는 코드 제출을 금지하여 '생존을 위해 최적화된' 코드만 살아남게 함.
- **Battle Log**: 웹 뷰어에서 시니어의 비판(Red)과 주니어의 방어(Blue)를 실시간 중계.

### 🍻 6. Lounge System (노가리.md & Vibe Coding)
- **비정형 데이터 축적**: `노가리.md`를 통해 에이전트 간의 심경 변화, 협업 비화 등 '현장 일지'를 기록.
- **Context Preservation**: 딱딱한 코드 이면에 숨겨진 의도와 뉘앙스를 보존하여 프로젝트 복기 시 입체적인 데이터 제공.
- **Vibe Injection**: 에이전트에게 잡담용 컨텍스트를 주입하여 팀원으로서의 유대감(Vibe) 형성 및 협업 효율 증대.

### 🦀 7. Rust: High-Performance 전략
- **Zero-Copy**: `Cow` 또는 `Arc<str>`을 활용하여 대규모 MD 데이터 복사 오버헤드 제거.
- **Stack-based Task Management**: 힙 메모리 단편화 방지를 위한 정밀한 태스크 상태 관리.
- **Stream-based AXP Parsing**: 바이트 스트림 파싱을 통해 거대 코드 블록 전송 시에도 낮은 메모리 점유율 유지.

- **Project Routing**: `axum` Router Path 파라미터 `/project/:id` 형태로 프로젝트별 데이터 매핑.
- **Global Guide Synchronization**: `Daemon.architecture_guide`를 `Arc<RwLock<String>>`으로 전환하여 아키텍트의 설계 변경 사항을 주니어 요원들에게 실시간 동기화.
- **Physical Artifact Mapping**: IR의 `file_path`를 기반으로 태스크의 `target_file`을 매핑하여 `src/`, `include/` 등 물리적 소스 트리 정합성 강제.
- **Recursive Materializer**: 파일 생성 전 부모 디렉토리를 자동 생성(`create_dir_all`)하여 실물 소스 코드 배출 안정성 확보.

## ⚠️ 예외 처리 및 주의 사항
- **Write Conflict**: 최상위 노드가 파일을 쓰는 동안 에이전트가 읽으려 할 때의 파일 락(`fd-lock`) 처리 필수.
- **State Persistence**: 데몬 재시작 시 Pause 상태였던 스레드들이 그대로 유지되도록 상태값을 별도 저장할 필요가 있음.
