# Axon v0.0.28 — Reliability Recovery & IR Integrity Goals

## Core Objective

Axon v0.0.28 aims to transform the current pipeline from:

```text
Fail-Open Autonomous Generation
```

into:

```text
Constraint-Grounded Deterministic Generation
```

The primary mission of v0.0.28 is not feature expansion.

It is:

```text
pipeline stabilization
```

through:

- IR integrity enforcement
- semantic validation
- hallucination containment
- language-aware decomposition
- hard-stop execution safety

---

# Primary Goal

Prevent invalid Skeleton IR from contaminating downstream stages.

Current failure pattern:

```text
spec.md
→ invalid Skeleton IR
→ corrupted architecture.md
→ hallucinated tasks
→ stub generation
→ invalid project output
```

Target behavior:

```text
spec.md
→ validated Skeleton IR
→ verified architecture.md
→ grounded task decomposition
→ language-aware implementation
→ stable project output
```

---

# v0.0.28 Success Definition

The release is considered successful only if:

```text
An invalid IR can never silently continue through the pipeline.
```

This is the central invariant of v0.0.28.

---

# Design Philosophy

v0.0.28 prioritizes:

```text
Correctness over continuity
```

Meaning:

```text
System halt is preferable to hallucinated execution.
```

The system must:

- refuse corruption
- reject invalid state
- stop unsafe execution
- preserve semantic integrity

instead of attempting uncontrolled fallback generation.

# Strategic Goals

# 1. Skeleton IR Integrity

## Objective

Guarantee that Skeleton output is semantically meaningful.

## Requirements

Skeleton IR must contain:

- valid components
- valid file paths
- meaningful functions
- language-aware structure
- spec-correlated decomposition

Invalid examples:

```json
{
  "id": "..",
  "path": ".."
}
```

must become unrecoverable errors.

---

# 2. Hard-Stop Enforcement

## Objective

Eliminate fail-open execution behavior.

Current:

```text
FAIL → continue
```

Target:

```text
FAIL → stop
```

## Mandatory Abort Conditions

Pipeline must stop if:

- architecture.md is invalid
- component graph is empty
- semantic overlap falls below threshold
- language constraints are violated
- placeholder leakage is detected

---

# 3. Language-Aware Architecture IR

## Objective

Enable native decomposition for compiled languages.

Current limitation:

```text
generic app-oriented IR
```

Target:

```text
language-native IR modeling
```

---

The IR must support:

```text
- translation units
- header/source relationships
- include graphs
- symbol exports
- linker boundaries
- compilation ownership
```

```json
{
  "component_type": "translation_unit",
  "header": "calculator.h",
  "source": "calculator.c",
  "exports": [],
  "includes": []
}
```

All downstream execution depends on IR correctness.

Therefore:

```text
invalid IR == invalid reality
```

The system must never continue from corrupted IR.

---

Current system behavior:

```text
detect corruption
but continue execution
```

v0.0.28 rejects this philosophy.

Target behavior:

```text
detect corruption
→ enforce halt
```

Different languages require different decomposition models.

# v0.0.28 — Phase 2: IR Addressability & Retry Loop Fix

## 문제 정의

TEST2 실행 결과 확인된 핵심 문제들:

### 1. IR Lookup Failure Pattern
```
Task: validators.c
IR Section: (nonexistent)
Result: [IR_ADDR_FAIL] Could not find IR section for validators.c
→ 무한 재시도 루프
```

### 2. 원인 분석
- IR Schema: 이미 symbol-aware (`Component.functions`)
- Orchestration: file-centric 작동
- IR Section Naming: `validators.c` 미등록 (`.h`만 존재 가능성)
- Task Decomposition: `task_validators_module` vs `validators.c` drift

### 3. Entropy Cascade 구조
```
CHUNK_FAIL → 전체 architecture guide fallback
→ Context Inflation
→ Degraded Retry
→ Malformed Patch
→ Rework Expansion
→ Positive Feedback Collapse Loop
```

### STEP 2: Retry Fuse + Path Normalization (진행 예정)
#### 2-1. Retry Fuse
- **목적**: deterministic failure의 무한 루프 방지
- **위치**: `crates/axon-daemon/src/lib.rs` - `spawn_rework_task`
- **구현**:
  ```rust
  const MAX_TASK_RETRIES: u32 = 3;

  if task.rework_count >= MAX_TASK_RETRIES {
      task.state = TaskState::Quarantined;
      error!("[TASK_QUARANTINED] task={} reason={}", task.id, last_error);
      return Ok(());  // queue에 재투입 금지
  }
  ```

#### 2-2. Path Normalization
- **파일**: `crates/axon-core/src/ir.rs`
- **함수**: `canonical_ir_name()`
- **구현**:
  ```rust
  pub fn canonical_ir_name(path: &str) -> String {
      let filename = path.split('/').last().unwrap_or(path);
      filename
          .trim_end_matches(".c")
          .trim_end_matches(".h")
          .trim_end_matches(".cpp")
          .trim_end_matches(".hpp")
          .to_lowercase()
  }
  ```
- **적용 위치**:
  - IR registration 시 (generator)
  - IR lookup 시 (resolver)

#### 2-3. Planner Validation Gate
- **목적**: invalid task의 queue 진입 차단
- **구현**:
  ```rust
  pub fn validate_task_ir_binding(task: &Task, ir: &ProjectIR) -> Result<()> {
      let target = canonical_ir_name(&task.target_file);
      if !ir.sections.contains_key(&target) {
          bail!("[INVALID_TASK_BINDING] task={} target={}", task.id, target);
      }
      Ok(())
  }
  ```

#### 2-4. IR Generator Fix
- **문제**: `.c/h` 표기 시 한 파일만 등록되는 현상
- **해결**: 두 파일 모두 IR에 등록하거나 대응 `.c` 빈 섹션 생성
