# Axon v0.0.28 — Skeleton Hard Stop + Semantic Validator

## Overview

The first priority of Axon v0.0.28 is not better generation.

It is:

```text
pipeline contamination prevention
```

The system currently suffers from a catastrophic architectural flaw:

```text
Invalid Skeleton IR is allowed to propagate downstream.
```

This causes:

- corrupted architecture.md
- hallucinated task generation
- unrelated project decomposition
- stub-only outputs
- semantic collapse

The solution begins with:

```text
Skeleton Hard Stop
+
Semantic Validator
```

These two systems form the first real enforcement layer in Axon.

---

# Why This Matters

Current pipeline:

```text
spec.md
→ Skeleton
→ architecture.md
→ HeaderGen
→ TaskGen
→ Implementation
```

All downstream stages assume:

```text
architecture.md == truth
```

This means:

```text
Skeleton = compiler front-end
```

If Skeleton IR becomes corrupted:

```text
all downstream stages inherit corruption
```

---

# Current Critical Failure

Observed behavior:

```text
Skeleton = FAILING
HeaderGen = HEALTHY
```

This means:

```text
The system detects corruption
but still continues execution.
```

This creates:

```text
fail-open autonomous hallucination
```

behavior.

---

# v0.0.28 Immediate Goal

Prevent invalid IR from ever reaching downstream stages.

Target behavior:

```text
Invalid Skeleton IR
→ immediate pipeline termination
```

instead of:

```text
Invalid Skeleton IR
→ hallucinated continuation
```

---

# Part 1 — Skeleton Hard Stop

# Objective

Create deterministic structural enforcement for Skeleton IR.

This layer must reject:

- malformed IR
- placeholder leakage
- empty architecture graphs
- invalid component definitions

before any downstream execution occurs.

---

# Core Principle

```text
Invalid structure = fatal error
```

No recovery.

No fallback generation.

No continuation.

---

# Required Structural Checks

## 1. Component Count Validation

Reject:

```python
if component_count == 0:
    reject()
```

---

## 2. Placeholder Detection

Reject:

```python
if component.path == "..":
    reject()

if component.name == "..":
    reject()
```

Known dangerous outputs:

```text
".."
"TBD"
"placeholder"
"example"
```

must become fatal.

---

# 3. Empty Function Validation

Reject:

```python
if len(component.functions) == 0:
    reject()
```

---

# 4. Invalid Path Validation

Reject:

```python
- malformed paths
- unsupported extensions
- illegal file names
```

---

# 5. Duplicate Symbol Validation

Reject:

```python
duplicate exports
duplicate component IDs
duplicate ownership
```

---

# Hard Stop Behavior

Current behavior:

```text
FAIL → continue
```

Target behavior:

```text
FAIL → ABORT PIPELINE
```

This is mandatory.

---

# Part 2 — Semantic Validator

# Objective

Ensure generated IR remains semantically connected to the original specification.

This validator exists because:

```text
valid JSON can still be hallucinated
```

Example:

```json
{
  "path": "frontend.tsx"
}
```

This is structurally valid.

But if the specification defines:

```text
C calculator project
```

then the IR is semantically invalid.

---

# Core Principle

```text
Structure alone is insufficient.
```

Semantic continuity must be enforced.

---

# Semantic Drift Problem

Current issue:

```text
spec.md
→ generic software prior
→ unrelated task generation
```

This creates:

```text
semantic drift
```

The Semantic Validator exists to stop this.

---

# Minimum Required Semantic Checks

# 1. Keyword Overlap Validation

Compare:

```text
spec keywords
vs
IR keywords
```

Example:

Specification:

```text
calculator
validator
age
header
c
```

IR:

```text
frontend
deployment
react
typescript
```

Result:

```text
low overlap
→ reject
```

---

# 2. Language Constraint Validation

If specification declares:

```text
Language: C
```

then reject:

```text
.tsx
package.json
npm
React
Node.js
```

This prevents language contamination.

---

# 3. Artifact Expectation Validation

For C/C++ projects:

Expected artifacts:

```text
.h
.c
```

If missing:

```text
reject
```
