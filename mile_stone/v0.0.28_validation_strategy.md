# Axon v0.0.28 — Validation Strategy & Verifiable Pipeline Roadmap

## Overview

The primary challenge facing Axon is no longer raw generation capability.

The real challenge is:

```text
verification
```

The current system can generate outputs, but it cannot reliably determine whether those outputs remain semantically connected to the original specification.

As a result:

```text
hallucinations propagate silently
```

through the pipeline.

v0.0.28 defines the transition from:

```text
LLM-centric generation
```

to:

```text
validation-centric autonomous compilation
```

---

# Core Objective

Transform Axon into a system where:

```text
Only validated state transitions are trusted.
```

The LLM must become:

```text
proposal engine
```

not:

```text
source of truth
```

The validator becomes the authority.

---

# Current Failure Pattern

Observed pipeline behavior:

```text
spec.md
→ invalid Skeleton IR
→ corrupted architecture.md
→ hallucinated task graph
→ stub generation
→ invalid project output
```

The root problem:

```text
generation without enforcement
```

---

# Target Pipeline

Desired future behavior:

```text
spec.md
→ validated IR
→ verified architecture
→ constrained decomposition
→ executable verification
→ trusted output
```

---

# Strategic Roadmap

# Phase 1 — Structural Validation

## Objective

Prevent structurally invalid IR from entering the pipeline.

Current problem:

```text
Malformed IR is accepted as long as JSON exists.
```

Target behavior:

```text
Invalid structure becomes fatal.
```

---

## Required Validations

### Component Validation

Reject if:

```text
- component_count == 0
- path == ".."
- empty function list
- duplicate symbols
- invalid file extensions
```

---

## Language Validation

For C/C++:

```text
- .h/.c pair existence
- translation unit consistency
- include ownership validation
```

must exist.

---

## Key Principle

Structural validation must be:

```text
100% deterministic
```

No LLM involvement.

---

# Phase 2 — Semantic Validation

## Objective

Verify that IR remains semantically connected to the specification.

This phase addresses:

```text
semantic hallucination
```

---

# Problem Example

Specification:

```text
calculator.c
validators.h
age calculation
```

Generated IR:

```text
frontend.tsx
deployment.py
```

This must become impossible to pass.

---

## Required Mechanisms

# 1. Keyword Overlap

Compare:

```text
spec keywords
vs
IR keywords
```

Low similarity:

```text
→ reject
```

---

# 2. Embedding Similarity

Use semantic embeddings to compare:

```text
spec ↔ architecture
spec ↔ tasks
architecture ↔ tasks
```

This creates semantic continuity verification.

---

# 3. Constraint Inheritance

Constraints defined upstream must survive downstream stages.

Example:

```text
Language: C
```

If downstream stages generate:

```text
React
Node.js
TypeScript
```

then:

```text
hard reject
```

---

# Phase 3 — Language-Aware Compilation Graph

## Objective

Replace generic file generation with compiler-aware decomposition.

Current system behavior:

```text
file generation pipeline
```

Target behavior:

```text
compilation graph pipeline
```

---

# Required IR Concepts

## C/C++

The IR must understand:

```text
- translation units
- header/source ownership
- include graphs
- exported symbols
- linker boundaries
- compile dependencies
```

---

# Example IR

```json
{
  "source": "calculator.c",
  "header": "calculator.h",
  "exports": [
    "calculate_age"
  ],
  "depends_on": [
    "validators.h"
  ]
}
```

---

# Why This Matters

This enables deterministic detection of:

```text
- missing implementations
- orphan headers
- unresolved symbols
- stub-only outputs
```

---

## Objective

Ensure every stage references the same semantic reality.

Current issue:

```text
Each stage hallucinates independently.
```

This creates:

```text
Skeleton reality
≠
HeaderGen reality
≠
TaskGen reality
```

---

# Required Solution

All stages must reference:

```text
shared immutable IR snapshot
```

# Required Validators

## Diff Validator

Reject if:

```text

```

```text

```

Reject if:

