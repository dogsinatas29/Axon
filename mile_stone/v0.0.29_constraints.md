# AXON v0.0.29 Constraint Specification
## Deterministic Contract-Driven Build Constraints

---

# 0. Objective

본 문서는 AXON v0.0.29 결정론적 코드 합성 파이프라인을 안정적으로 유지하기 위한 제약 조건(Constraints) 정의 문서다.

핵심 목표는 다음과 같다.

- 확률론적 생성 붕괴 방지
- signature drift 차단
- hallucinated symbol 제거
- deterministic orchestration 보장
- weak model 환경 안정화
- compile-success gating 강제

---

# 1. Constraint Philosophy

## 1.1 Determinism over Creativity

LLM의 자유 생성보다:
- 제한된 계약
- 고정된 인터페이스
- 최소 수정 범위
를 우선한다.

---

## 1.2 Contracts are Immutable

검증 및 Freeze된 인터페이스는 수정 불가능하다.

대상:
- function signatures
- public symbols
- include ownership
- extern declarations

---

## 1.3 Orchestration is the True Authority

LLM은 worker다.

실제 지능은:
- orchestration layer
- registry authority
- compile gates
- dependency scheduling
에 존재한다.

---

# 2. Structural Constraints

## SC-001 — Header-first Rule

```text
Implementation generation is forbidden
until all headers are validated and frozen.
```

목적:
- dependency-blind generation 차단
- interface drift 방지

---

## SC-002 — Main-last Rule

```text
main.c can only be generated
after all implementation units compile successfully.
```

목적:
- premature integration 방지
- phantom symbol 생성 차단

---

## SC-003 — Task Count Integrity

```text
All implementation tasks declared in IR
must exist before ImplGen starts.
```

목적:
- missing implementation 파일 방지

---

## SC-004 — Include DAG Integrity

```text
Cyclic include graphs are forbidden.
```

목적:
- include recursion 차단
- compile deadlock 방지

---

## SC-005 — Ownership Isolation

```text
Agents may only modify owned files.
```

목적:
- repair cascade 차단
- unrelated mutation 방지

---

# 3. Symbol Constraints

## SYM-001 — Registry-only Symbols

```text
Only symbols registered in Symbol Registry
may be referenced.
```

목적:
- hallucinated APIs 차단
- phantom functions 제거

---

## SYM-002 — Frozen Signatures

```text
Frozen function signatures are immutable.
```

목적:
- signature drift 차단

---

## SYM-003 — Include Whitelist

```text
Only approved headers may be included.
```

목적:
- illegal include 차단

---

## SYM-004 — No Phantom APIs

```text
Undefined functions are forbidden.
```

목적:
- linker failure 방지

---

## SYM-005 — Explicit Globals Only

```text
Unregistered global state creation is forbidden.
```

목적:
- hidden coupling 차단

---

# 4. Behavioral Constraints

## BEH-001 — Stub Prohibition

```text
Empty implementations are forbidden.
```

금지 대상:
- empty bodies
- TODO stubs
- placeholder logic

---

## BEH-002 — Placeholder Detection

탐지 대상:
- TODO
- FIXME
- Hello World
- return 0 only
- dummy implementations

목적:
- deterministic garbage 차단

---

## BEH-003 — Contract Assertions

```text
Implementation outputs must satisfy
declared contract assertions.
```

목적:
- semantic correctness 최소 보장

---

## BEH-004 — No Silent Failure

```text
Error paths must be explicit.
```

목적:
- hidden failure 차단
- unsafe fallback 방지

---

# 5. Orchestration Constraints

## ORCH-001 — Compile Gate Blocking

```text
Compilation failure blocks downstream tasks.
```

목적:
- cascading failure 차단

---

## ORCH-002 — Retry Bound

```text
Repair retries are capped.
```

기본값:
- max_retry = 3

목적:
- infinite repair loop 방지

---

## ORCH-003 — Minimal Repair Scope

```text
Repair may only modify local symbol scope.
```

목적:
- repair hallucination cascade 차단

---

## ORCH-004 — Deterministic Queue Ordering

```text
Task execution order must be deterministic.
```

순서:
1. Headers
2. Implementations
3. Integration(main.c)

---

## ORCH-005 — Immutable Build Context

```text
Frozen artifacts may not be mutated downstream.
```

목적:
- build drift 차단

---

# 6. AST-Level Validation Rules

컴파일 이전 lightweight validation 수행.

탐지 대상:
- unknown symbols
- illegal includes
- duplicate definitions
- invalid ownership
- forbidden globals

목표:
- compiler invocation 비용 감소
- repair latency 감소

---

# 8. Final Principle

AXON의 우선 목표는:
- 완벽한 semantic intelligence
가 아니다.

우선 목표는:
- deterministic orchestration
- contract integrity
- compile stability
- reproducible synthesis

이다.

Weak model 환경에서는:
- 자유 생성보다
- 제한된 계약 기반 합성이 더 높은 안정성을 만든다.

The orchestration layer is the real compiler.
