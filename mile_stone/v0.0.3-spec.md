
# AXON Specification
## v0.0.3-spec

### Purpose
Define the initial architectural direction of AXON based on the rough spec and scenario.

### Core Concept
AXON is a virtual AI development team runtime that performs software development through structured roles and review pipelines.

Roles:
- Junior Agent (implementation)
- Senior Agent (review)
- Architect Agent (architecture integrity)
- User (final authority)

### Development Flow

Spec
↓
Architect
↓
Task Decomposition
↓
Junior Coding
↓
Review
↓
User Approval

### Diff‑based Review

AXON never applies changes directly.

Instead it generates a diff between:

project source
and
sandbox workspace

The user reviews this diff before accepting changes.

### Approve & Lock

When a change is accepted:

APPROVE & LOCK

occurs and the change becomes part of the official architecture.
