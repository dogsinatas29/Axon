## Initial Risk Rule Categories

### 1. Undefined Data Model

Trigger Conditions:

* unknown struct usage
* opaque pointer without contract
* field access on undefined type

Example:

```c id="34cbcr"
struct user_record user;
```

Without sealed layout definition:

```text id="h2icxg"
→ SEMANTIC_INTERRUPT
→ BLOCKING
```

---

### 2. Ownership Ambiguity

Trigger Conditions:

* malloc without ownership contract
* pointer return without free policy
* sqlite3 result lifetime undefined

Example:

```c id="x0g3r4"
char* get_user();
```

Without:

```text id="u1w6lc"
ownership_policy
```

Result:

```text id="ixaw0q"
→ GENERATION_DENIED
```

---

### 3. Dependency Escalation

Trigger Conditions:

* optional dependency promoted to blocking
* external library introduced without linkage policy
* unresolved include contract

Example:

```c id="bn23ha"
#include <ncurses.h>
```

Without:

```text id="g7j1zi"
dependency_policy
```

Result:

```text id="j0v0eu"
→ INTERRUPT
```

---

### 4. ABI Drift Detection

Trigger Conditions:

* header/source signature mismatch
* architecture IR mismatch
* symbol naming divergence

Example:

```c id="0v2r3o"
init_db()
```

vs

```c id="n4m3sx"
init_database()
```

Result:

```text id="rkl2em"
→ HARD BLOCK
```

---

### 5. Lifecycle Ambiguity

Trigger Conditions:

* init/shutdown pair missing
* resource cleanup undefined
* state transition incomplete

Example:

```c id="6m8v6d"
sqlite3_open()
```

without guaranteed:

```c id="4j7x5x"
sqlite3_close()
```


