# ⚖️ Semantic Sealing

## Goal

Prevent LLMs from inferring undefined contracts.

The engine must never invent:

* data structures
* ownership rules
* lifecycle behavior
* dependency policies
* ABI/linkage contracts

---

# 🚨 Semantic Risk Matrix

| Domain      | Interrupt Trigger                   | Sealed Contract                |
| ----------- | ----------------------------------- | ------------------------------ |
| Data Model  | Undefined struct usage              | field names, types, alignment  |
| Ownership   | Pointer/free responsibility unclear | caller/callee ownership        |
| Dependency  | Optional dependency policy missing  | mandatory / optional / exclude |
| ABI/Linkage | Symbol visibility ambiguous         | static / extern / header scope |
| Lifecycle   | Init/shutdown order undefined       | state transitions & teardown   |

---

# 🧠 Semantic Debugger Stage

## Purpose

The Semantic Debugger is NOT a reasoning engine.

Its role is strictly limited to:

### Allowed

* ambiguity detection
* unresolved contract extraction
* ownership gap detection
* dependency policy detection
* lifecycle ambiguity detection
* optional escalation detection
* ABI mismatch detection

### Forbidden

* automatic API invention
* automatic struct generation
* ownership inference
* dependency guessing
* architecture synthesis

Core rule:

```text
Detect only.
Never invent.
```
