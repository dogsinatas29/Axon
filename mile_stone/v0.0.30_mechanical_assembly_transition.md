# 🧩 Semantic Closure Engine

## Purpose

The Semantic Debugger is NOT a reasoning engine.

It exists ONLY to detect unresolved contracts.

Allowed responsibilities:

- ambiguity detection
- ownership gap detection
- dependency escalation detection
- ABI mismatch detection
- lifecycle contract detection
- unresolved struct usage
- semantic risk extraction

Forbidden responsibilities:

- API invention
- automatic struct generation
- ownership inference
- dependency guessing
- architecture synthesis
- speculative repair

Core doctrine:

```text
Detect only.
Never invent.
```
