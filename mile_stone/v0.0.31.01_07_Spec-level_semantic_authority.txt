2. 아직 비어 있는 핵심 계층

아직 없는 것:

Spec Semantic Validation Layer

이게 핵심이다.

현재는:

spec 자체의 semantic worldview

를 검증하지 않는다.

3. 지금 구조의 실제 문제

현재 흐름:

Spec
↓
LLM JSON
↓
RawIR
↓
Semantic Firewall

문제:

RawIR projection 자체가 이미 semantic contamination 가능

이라는 점이다.

즉:

semantic firewall은 너무 늦다
4. 실제로 지금 못 막는 것

예:

{
  "language": "rust",
  "components": [
    {
      "name": "header",
      "kind": "HeaderDecl"
    }
  ]
}

현재 구조에서는:

JSON parse 성공
RawIR projection 시도
이후 artifact legality 검사

즉:

semantic intent contamination

자체는 이미 발생한다.

semantic worldview legality

즉:

vocabulary legality

build-system legality

검사해야 한다.

(3) RawIR 이전 validation

핵심.

현재:

목표:

RawIR 생성 전 reject
6. 가장 중요한 추가 작업
Language Resolution 선행

language declared first

즉:

가 제일 먼저 와야 한다.
