v0.0.31.01 이후 필요한 실제 작업 구조

지금부터 필요한 건 단순 validator 추가가 아니다.

핵심은:

Spec → Semantic Universe Binding

을 시스템의 최상단으로 끌어올리는 것이다.

즉:

IR 생성 이전

에 semantic authority를 완성해야 한다.

1. 가장 먼저 해야 하는 작업
SemanticOntology를 먼저 formalize

이게 루트다.

현재는:

LanguageProfile

만 존재한다.

하지만 필요한 건:

언어별 semantic law definition

이다.

2. 새 디렉토리 필요

예상 구조:

crates/axon-ir/src/semantic/
3. SemanticOntology 정의

핵심 타입.

예:

pub trait SemanticOntology {
    fn allowed_artifacts(&self) -> &[ArtifactKind];

    fn forbidden_vocabulary(&self) -> &[&str];

    fn allowed_task_kinds(&self) -> &[LanguageTaskKind];

    fn allowed_build_systems(&self) -> &[BuildSystem];

    fn topology_rules(&self) -> TopologyRules;
}
4. 언어별 Ontology 구현

필수:

RustOntology
COntology
PythonOntology
5. 여기서 중요한 점

지금의:

allows_artifact()

는 너무 약하다.

왜냐면 현재는:

physical artifact legality

만 보기 때문이다.

하지만 ontology는:

semantic worldview legality

를 정의해야 한다.

6. 그 다음 작업
SpecSemanticValidator 도입

새 구조:

semantic/spec_validator.rs
7. SpecSemanticValidator 역할

이 계층은:

LLM output meaning inspection

을 담당한다.

즉:

vocabulary legality
topology intent
decomposition legality
build ontology legality

검사한다.

8. 여기서 가장 중요한 점

현재 validator는:

IR structural validation

이다.

SpecSemanticValidator는:

pre-IR semantic intent validation

이다.

완전히 다른 계층이다.

9. 새 Validation 계층 구조 필요

최종적으로:

SpecSemanticValidator
↓
IRValidator
↓
ArtifactFirewall

3단계가 된다.

직행한다.

이걸 바꿔야 한다.

13. extracted_terms 중요

왜냐면 contamination은:

semantic vocabulary

에서 발생하기 때문이다.

예:

HeaderDecl
CMakeLists.txt
translation unit

같은 단어.

14. SemanticViolation hierarchy 필요

현재 ValidationError는 너무 generic하다.

새 계층:

pub enum SemanticViolation {
    ForbiddenVocabulary,
    CrossLanguageContamination,
    InvalidTopologyIntent,
    InvalidBuildSystem,
    InvalidTaskVocabulary,
}

syntax repair

semantic worldview correction

이다.
