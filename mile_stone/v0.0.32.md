## 🧠 상세 설계 및 로직
### 1. Tiny Mutation Corpus 확대 (10~20개 사례)
- **대상:** GTK, Win32, Legacy Async, Plugin Lifecycle, Timer Cleanup, Queue Stabilization, Weak Ownership, Retry Cancellation.
- **목적:** 실제 레거시 핫스팟(Brownfield)에서 발생하는 작은 위상 변이(Small Real Mutation)를 집중 수집. 기능 개선이나 성능 향상이 아닌 **"위상 보존(Topology Preservation)"**에만 집중.

### 2. SAFE / REJECT 사례의 균형 축적
- AXON 커널의 본질은 "위험한 변이를 정확히 기각하는 것".
- 안전한 진화(Safe Evolution)와 위험한 진화(Unsafe Drift)를 교차 주입하여 `TOPOLOGY_DRIFT_REJECTED`와 `SAFE_TO_MERGE` 판결의 정밀도를 동시에 입증.

### 3. Collapse Lineage Taxonomy (병리 분류 체계) 안정화
- **현재 관측된 병리들:**
  - `DeferredOrphanDispatch`
  - `RecursiveEmitStorm`
  - `StaleHWNDDispatch`
  - `OrphanedAsyncTask`
- **목적:** 위 병리들이 실증 과정에서 통폐합(Merge/Split)될 가능성이 크므로, CI 스키마로 고착(Premature Freeze)시키기 전에 실측 데이터를 통해 분류 체계를 숙성.

### 4. Replay Runtime Cost Profiling (운영 지연 한계 측정)
- 리플레이 기반 증명의 가장 큰 치명타는 **시간 비용**.
- 리플레이 1회에 15분이 소요된다면 실무에서 외면당하므로, `Operational Latency Ceiling`을 측정하고 한계를 설정.
