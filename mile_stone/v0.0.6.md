
# AXON Specification
## v0.0.6

### Core Concept

AXON의 핵심 엔진은 두 가지 시스템으로 구성된다.

- Thread Engine
- Nogari Engine

이 두 시스템은 동일한 기반 구조를 공유하며 모두 **Thread 기반 시스템**으로 동작한다.

---

# 1. Thread Engine

Thread는 AXON에서 **작업의 기본 단위**이다.

Thread는 하나의 개발 작업, 기능 구현, 리팩토링 또는 문제 해결을 의미한다.

예시

Thread-21
title: websocket reconnect 구현
worker: Junior
reviewer: Senior
status: running

Thread는 다음과 같은 상태를 가진다.

draft
queued
running
review
approved
merged
archived

---

# 2. Spec Processing Model

AXON은 명세서를 내부 파서로 처리하지 않는다.

대신 **LLM을 Spec Parser로 사용한다.**

흐름

Spec Document
↓
LLM Parsing
↓
Task Decomposition
↓
Thread 생성

LLM은 다음 작업을 수행한다.

- 명세 이해
- 작업 분해
- 작업 우선순위 판단
- 작업자 할당

이 결과는 **architecture.md**에 기록된다.

---

# 3. Worker Assignment

작업자는 다음 구조를 따른다.

Architect
↓
Senior
↓
Junior

기본 작업 흐름

Junior → 구현
Senior → 코드 리뷰
Architect → 아키텍처 검증
User → 최종 승인

단, 동일 작업에 대해 **중복 Thread 생성이 허용된다.**

예

Thread-31  (Junior-A)
Thread-32  (Junior-B)

Senior가 최종 해결책을 선택한다.

---

# 4. Architecture Record

AXON의 공식 작업 기록은 **architecture.md**에 저장된다.

이 문서는 다음 정보를 포함한다.

- Thread 목록
- 작업 제목
- 담당 Agent
- 리뷰 담당자
- 작업 상태
- 설계 결정

예

Thread-21
title: websocket reconnect
worker: Junior
reviewer: Senior
status: running

---

# 5. Nogari Engine

Nogari는 AXON의 **토론 및 설계 논의 시스템**이다.

Nogari는 개발팀의 회의나 설계 토론을 모델링한다.

목적

- 설계 논의
- 문제 분석
- 코드 리뷰 토론
- 아키텍처 검증

예

Junior: reconnect 로직 구현했습니다.
Senior: timeout 처리가 빠졌습니다.
Architect: reconnect 정책을 수정해야 합니다.

---

# 6. Nogari Thread Model

Nogari는 **Thread Engine과 동일한 구조**를 사용한다.

즉 Nogari 또한 **Thread 기반 시스템**이다.

구조

Nogari Thread
 └ Discussion Messages

이 구조는 일반 작업 Thread와 동일한 엔진 위에서 동작한다.

---

# 7. LLM Generated Threads

AXON에서는 **LLM이 직접 Thread를 생성할 수 있다.**

이 기능은 v0.0.3 명세에서 정의된 기능이다.

예

LLM 분석 결과

Spec → 새로운 작업 필요

LLM은 다음을 수행할 수 있다.

Thread 생성
title: reconnect retry policy 개선
worker: Junior

또는

Nogari Thread 생성
topic: reconnect 정책 설계 논의

---

# 8. Unified Thread System

AXON의 Thread 시스템은 두 가지 용도로 사용된다.

1. Development Thread
2. Nogari Thread

구조적으로 동일하지만 목적이 다르다.

Development Thread

코드 구현 작업

Nogari Thread

설계 논의 및 토론

---

전체 시스템 흐름

---

# 10. Design Philosophy

AXON의 개발 철학

AI writes code
Human owns code

또한

Discussion drives architecture.

즉 AXON은 **Thread 기반 개발 + 토론 기반 설계 시스템**이다.
