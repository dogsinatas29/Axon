📅 작업 정보
상태: 🏗️ Planned

관련 마일스톤: Initial Design

목표: 로컬 데몬 기반의 멀티 에이전트 개발 환경 자동 구축

🧠 상세 설계 및 로직
1. 서비스 엔트리포인트 (Daemon & Init)
Daemon Mode: Rust 기반의 axond가 백그라운드에서 프로젝트 상태를 감시.

Context Capture: axon init 실행 시 현재 디렉토리의 메인 MD(예: 요구사항 명세)를 분석하여 ARCHITECTURE_AXON.md를 자동 생성. 이 파일이 전체 프로젝트의 설계 바이블이 됨.

Sanitization Layer: axond가 파일을 읽기 전, 모든 문자열에서 \u200B 같은 제어 문자를 strip 처리.

Safety Lock: 유효하지 않은 UTF-8 바이트가 발견되면 "Invalid Character in [Filename]" 로그를 찍고 게시판에 '시니어' 페르소나로 "이보게, 파일명에 쓰레기가 섞였군."이라고 일침을 날리도록 설계.

2. 에이전트 위계 및 페르소나 주입 (Hierarchy & Persona)
Scale Out: 사용자가 n명의 주니어를 지정하면, 각 에이전트별로 전용 워크스페이스(MD 파일)가 할당됨.

Persona Injection: * Default: 표준화된 시니어/주니어 페르소나 자동 주입.

Custom: 사용자가 각 에이전트의 성격, 기술 스택, 엄격함을 직접 입력.

Role Definition: senior.md는 전체 로직 검토 및 승인을, junior_n.md는 개별 모듈 구현 및 유닛 테스트를 담당하도록 역할 분리.

3. 커뮤니케이션 레이어 (Bulletin Board & Humor)
Threaded Board: AXON 데몬이 로컬 웹 서버를 띄워 스레드형 게시판 활성화.

Humor Logic: '게시판 농담 빈도' 설정에 따라, 에이전트들이 작업 중간에 프로젝트 상황에 맞는 농담이나 '차도남' 스타일의 냉소적인 피드백을 출력. (예: "이 코드는 제 할머니가 짜도 이것보단 효율적일 것 같군요.")

🛠️ 기술적 변경 사항
Node Update: 에이전트 노드 생성 시 Role 필드(Senior/Junior) 및 HumorLevel 필드 추가.

Edge Update: ARCHITECTURE_AXON.md와 개별 에이전트 MD 간의 양방향 동기화 로직.

File Changes: - axon_config.json: 에이전트 수 및 페르소나 설정 저장.

generator/template.rs: 페르소나 및 기본 MD 구조 생성 템플릿.

⚠️ 예외 처리 및 주의 사항
컨텍스트 오염 방지: 주니어 에이전트들이 서로의 MD 파일을 동시에 수정하여 논리가 꼬이는 현상 방지 (File Locking 또는 순차적 처리).

무한 루프 방지: 농담 빈도가 너무 높을 경우 실제 작업 지시가 묻히지 않도록 UI 가중치 조절.
