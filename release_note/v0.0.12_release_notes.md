# 🚀 AXON Release Notes - v0.0.12

## 📝 개요
이번 버전에서는 AXON의 핵심 철학인 **"공정 제어 시스템으로서의 개발(SCADA for Software)"**을 완성하기 위한 이벤트 기반 아키텍처 고도화와 에이전트 계층 구조(Org Chart)의 완전한 통합이 이루어졌습니다.

## 🏛️ 주요 변경 사항

### 1. 이벤트 시스템 통합 및 직렬화 고도화 (Event Consolidation)
- **전역 이벤트 버스 정형화**: 모든 시스템 신호(Signal), 에이전트 응답(Response), 사장 명령(Boss Spec)이 단일 이벤트 스키마로 통합되었습니다.
- **이벤트 영속화**: `event_log` 테이블을 통해 과거의 모든 난상토론(Nogari)과 결정 사항이 기록됩니다.
- **Payload 확장**: JSON 기반 페이로드를 지원하여 복잡한 데이터 시트 전송이 가능해졌습니다.

### 2. 하드코어 에이전트 워크플로우 (Reactive Pipeline)
- **Junior -> Senior -> Architect 파이프라인**: 주니어의 제안을 시니어가 검토하고, 아키텍트가 최종 승인하는 3단계 검증 시스템이 백그라운드 데몬에 탑재되었습니다.
- **실시간 반응형 런타임**: 에이전트들이 이벤트 버스를 감시하며 본인의 역할에 맞는 신호가 오면 즉시 연산을 시작합니다.

### 3. 계층적 조직 관리 (Org Management & Succession)
- **Hiring/Firing API**: 웹 뷰어를 통해 실시간으로 에이전트를 고용하거나 해고할 수 있습니다.
- **계승 정책(Succession Policy)**: 관리자급 에이전트(Senior/Architect) 해고 시, 하위 에이전트들이 상위 관리자에게 자동 재배치되어 공정 중단을 방지합니다.
- **DTR (Tension Control)**: 에이전트별 긴장 상태 조절 기능을 위한 데이터 구조가 확정되었습니다.

### 4. 실시간 관제 센터 (Studio Dashboard)
- **WebSocket 라이브 스트리밍**: 팩토리 내부의 모든 시그널을 리프레시 없이 실시간으로 모니터링할 수 있습니다.
- **전투 로그(Battle Log) 시각화**: 에이전트 간의 논쟁과 협업 과정을 시각적으로 확인할 수 있는 기반을 마련했습니다.

## 🛠️ 기술적 세부 사항
- **Crates Update**: `axon-core`, `axon-storage`, `axon-daemon`, `axon-agent` 전반에 걸친 대규모 리팩토링 및 동기화.
- **API Expansion**: `/api/agents/hire`, `/api/agents/fire`, `/api/status` 등 관제용 엔드포인트 추가.
- **Frontend Refactoring**: React Context와 WebSocket Hook을 활용한 실시간 상태 동기화 로직 강화.

## ⚠️ 다음 작업 예정 (Roadmap)
- **Multi-Project Isolation UI**: 각 프로젝트별 독립 뷰어 및 라우팅 UI 완성.
- **Adversarial Persona Injection**: 에이전트 간의 더 격렬한 토론을 유도하는 페르소나 강화.
