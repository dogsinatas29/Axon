# 🚀 AXON v0.0.12 Release Notes - Control & Isolation

---

## 🛠️ 핵심 변경 사항 (Key Changes)

### 1. 🕹️ 실시간 작업 제어 (Pause / Resume)
- **메커니즘**: `tokio::sync::watch` 채널을 활용하여 데몬의 메인 오케스트레이션 루프를 즉시 일시 정지 및 재개하는 기능을 구현했습니다.
- **상태 전파**: 사용자가 웹 컨트롤 패널(또는 API)을 통해 신호를 보내면, 모든 하위 에이전트 스레드에게 `HOLD` 상태가 전파됩니다.
- **목표**: 토큰 소모를 효율적으로 관리하고, 사용자의 개입이 필요한 시점에 시스템을 안전하게 멈출 수 있습니다.

### 2. 🛡️ 프로젝트 격리 (Project Isolation)
- **데이터 구조**: `Thread` 및 `Task` 모델에 `project_id` 필드를 추가했습니다.
- **멀티 테넌시**: 하나의 데몬에서 여러 프로젝트를 동시에 관리하되, 데이터가 섞이지 않도록 스토리지 및 API 레벨에서 엄격하게 격리합니다.
- **API 확장**: `/api/project/:project_id/threads` 등 프로젝트별 엔드포인트를 신설하여 독립적인 관제가 가능해졌습니다.

### 3. 🔒 아키텍처 락인 (Architecture Lock-in)
- **성역화**: 확정된 코드 섹션을 `architecture.md`에 `[✅ Locked]` 마크업과 함께 박제하는 로직을 구현했습니다.
- **자동화**: BOSS(사용자)가 스레드를 최종 승인하는 즉시, 해당 내용이 아키텍처 문서에 반영되어 에이전트들이 이를 '변경 불가능한 성역'으로 인식하게 합니다.

### 4. 🗂️ 계층형 인사 관리 기초 (Org Chart Foundation)
- **에이전트 확장**: `Agent` 구조체에 `parent_id`, `dtr`, `name`, `model` 필드를 추가하여 조직 계층도를 그릴 수 있는 데이터를 확보했습니다.
- **인사 이동 API**: `/api/agents` 엔드포인트를 통해 현재 조직의 구성과 상태를 조회할 수 있습니다.
