# AXON v0.0.17 Release Notes - Control & Isolation

## 🚀 Key Features

### 1. Multi-Agent Orchestration (Phase 1-4)
- **Layer-based Execution**: Strictly enforces `Junior -> Senior -> Architect` chain of command.
- **Agent Pool Support**: Multiple models can be assigned to Junior and Senior layers.
- **Round-Robin Scheduling**: Stateless task distribution across available agents in each layer.
- **Hard Constraints**: Execution is blocked if any layer (Junior/Senior) lacks operational agents.

### 2. Ollama Runtime Adapter
- **Native Support**: Integrated Ollama API for local model orchestration.
- **Performance Tracking**: Captures `total_duration`, `eval_count`, and `eval_duration` from Ollama responses.

### 3. Observability & Reporting (Phase 5)
- **ObservabilityReport**: Detailed metrics collection for every execution step.
- **EventBus Integration**: Execution paths and failures are published in real-time to the EventBus.
- **Runtime Metrics**: Unified metric collection across different model drivers (Gemini, Claude, OpenAI, Ollama).

### 4. Robust Bootstrap Protocol
- **Phased Initialization**: Config validation, runtime adaptation, and context building are gated by strict Phase checks.

## 🛠️ Technical Changes
- **crates/axon-model**: Updated `ModelDriver` trait to return `ModelResponse` with metrics.
- **crates/axon-core**: Added `AgentValidation`, `ExecutionContext`, `ObservabilityReport`, and `RuntimeMetrics`.
- **crates/axon-daemon**: Implemented `handle_assignment` logic with layer checks and metric collection. Updated storage schema for persistent metrics.

## ⚠️ Known Issues
- Round-robin is currently based on task ID hash; may result in uneven distribution for small task sets.
- Architect fallback: If no juniors/seniors are found, the system currently fallback to Architect (as per v0.0.16) but Phase 4 constraints will eventually block it if not configured.
