# AXON v0.0.18 Release Notes

## 🇰🇷 한국어 (Korean)

### 🚀 핵심 성과 및 기능 (Key Achievements)
* **결과 생성 보장 (Output Generation Guarantee)**: 3-Tier 파서(Parser) 아키텍처가 완벽히 작동함을 입증했습니다. 표준 컨트랙트(Output Contract) 파싱이 실패하더라도, 마지막 단계인 휴리스틱(Heuristic) 파서가 개입하여 성공적으로 코드를 추출하고 파일로 생성합니다.
* **모델 교체 효과 입증 (Model Substitution)**: 로컬 모델을 `Mistral:latest`에서 `Gemma2`로 교체하여 테스트한 결과, Gemma2가 AXON의 출력 규약(Output Contract)을 훨씬 더 엄격하게 준수하며 안정적인 결과를 보장함이 확인되었습니다.
* **병목 방지 자동 위임 (Architect Bypass via Sampling Rate)**: 아키텍트(Architect)의 검토 부하를 줄이기 위한 `sampling_rate` 로직(예: 0.3)이 실전에서 완벽히 작동하여, 기준치를 초과할 경우 자동으로 시니어(Senior) 요원에게 승인 권한을 위임합니다.

### 🛠️ 치명적 버그 수정 (Critical Bug Fixes)
* **[CRITICAL] 0바이트 살인마 버그 (0-Byte File Overwrite Bug)**: 
  * **원인**: 데몬의 Semantic Merge 로직에서, 현재 태스크가 수정하지 않은 기존 파일(`new` 코드가 비어있는 상태)에 대해 병합 처리를 할 때 실수로 빈 문자열(`""`)을 반환하여 기존 정상 파일들을 0바이트로 덮어씌우는 치명적 결함이 있었습니다.
  * **해결**: 태스크가 파일을 수정하지 않았을 경우(빈 코드 반환 시) 기존 코드를 원형 그대로 보존(`return Some(current)`)하도록 로직을 수정하여 완벽하게 해결했습니다.
* **[CRITICAL] 구글 서버 과부하(503) 셧다운 방지 (Gemini 503 High Demand Protection)**:
  * **원인**: Gemini API 서버 과부하로 인해 503(High Demand) 에러가 발생할 경우, 시스템이 이를 치명적 오류로 간주하여 팩토리 가동을 중단하는 문제가 있었습니다.
  * **해결**: 해당 에러를 `QUOTA_WAIT` 상태로 변환하여 에이전트가 데몬을 종료시키지 않고 60초간 대기한 후 자동으로 재시도하도록 방탄 로직을 추가했습니다.
* **휴리스틱 파서 쓰레기 파일 추출 방지 (Heuristic Parser Garbage Exclusion)**:
  * 에이전트가 남긴 로그나 설명(`markdown`, `tool_code`, `bash` 등)을 휴리스틱 파서가 코드로 오인하여 `generated_*.tool_code` 파일로 추출하는 현상을 차단했습니다. 비코드(Non-code) 언어 블록은 무시하도록 필터링을 강화했습니다.

## 🇺🇸 English

### 🚀 Key Achievements
* **Output Generation Guarantee**: The 3-Tier Parser architecture is proven to work flawlessly. Even if the standard Output Contract parsing fails, the fallback Heuristic parser successfully intervenes, extracts the code, and creates the files.
* **Model Substitution Proven**: Swapping the local model from `Mistral:latest` to `Gemma2` demonstrated that Gemma2 adheres much more strictly to AXON's Output Contracts, ensuring more stable results.
* **Bottleneck Prevention (Architect Bypass)**: The `sampling_rate` logic (e.g., 0.3) successfully triggered in practice, bypassing the Architect's review to prevent bottlenecks and automatically delegating approval authority to the Senior agent.

### 🛠️ Critical Bug Fixes
* **[CRITICAL] 0-Byte File Overwrite Bug**: 
  * **Cause**: In the daemon's Semantic Merge logic, if the current task did not modify an existing file (i.e., `new` code was empty), the logic mistakenly returned an empty string (`""`), overwriting completely valid existing files with 0 bytes.
  * **Fix**: Patched the logic to explicitly preserve the existing code (`return Some(current)`) if the incoming task returns empty code, fully resolving the bug.
* **[CRITICAL] Gemini 503 High Demand Shutdown Protection**:
  * **Cause**: Spikes in Gemini API demand causing 503 (High Demand) errors were treated as fatal LLM Errors, shutting down the entire factory.
  * **Fix**: Added bulletproof logic to treat these specific 503 errors as a `QUOTA_WAIT` state, instructing the agent to pause for 60 seconds and automatically retry instead of crashing.
* **Heuristic Parser Garbage Exclusion**:
  * Prevented the heuristic parser from extracting non-code blocks (like `markdown`, `tool_code`, `bash` logs) as actual source code files (e.g., `generated_*.tool_code`). Added strict filters to ignore non-code markdown blocks.
