# 🚀 AXON Release Notes - v0.0.19

## [v0.0.19] - 2026-04-29

### 🛡️ Identity & Intelligence
- **Model Identity Sync**: Fixed a critical bug where agents were hard-coded as "Gemini 1.5 Pro". Now agents correctly recognize their identity from `axon_config.json` (e.g., Qwen 1.8B).
- **Intelligence-Aware Prompting**: Implemented a dual-track prompt system. Small models (like Qwen 1.8B) now receive an ultra-minimalist, one-shot template to prevent cognitive overload.

### ⚖️ Spec Drift Zero (IR Engine)
- **Constraint IR**: Introduced a Single Source of Truth (SSOT) for all rules. Prompts and Validators are now deterministically generated from a single IR object.
- **Conditional Forbidden Words**: Resolved the conflict between 'node' global forbidden rule and Mermaid dependency graphs. `node` is now allowed exclusively inside mermaid blocks.
- **Strict Contract Enforcement**: Ambiguous "Structural Warnings" have been replaced with hard-fail validation to ensure output integrity.

### 🧩 Profile Matrix
- **IR Modularity**: Rules are now grouped into reusable Profiles (e.g., `python_standard`, `strict_mermaid`).
- **Deterministic Merge**: Implemented a merging algorithm with clear priorities (`Global < Profile < Override`) and conflict detection.

### 🛠️ Technical Fixes
- Updated `AgentRuntime` to accept model names as a constructor argument.
- Updated `Daemon` to propagate model configuration to all workers.
- Refined Stage 1 bootstrapping validation to log failed outputs for easier debugging.
