# 🚀 v0.0.1 Release Notes - Language Enforcement Hardening

## 📅 날짜: 2026-05-06

## 🛠️ 변경 사항 (Changes)
### 1. 강력한 언어 준수 (Strong Language Enforcement)
- **대상**: `ko_KR` 로케일 설정 시의 모든 AI 에이전트 응답.
- **내용**: `Thought`, `Lounge(노가리)`, `Comments(주석)`, `Logs` 등 모든 텍스트 필드를 한국어로 고정.
- **구현**: 시스템 프롬프트의 가장 최상단에 `[LANGUAGE_ENFORCEMENT]` 섹션을 강제 삽입하여 Llama3 등의 모델이 영어나 중국어로 회귀하는 것을 방지.

### 2. 프롬프트 구조 통합
- `process_task` (Junior Agent Task)
- `generate_ir` (Architect IR Generation)
- `repair_ir` (IR Auto-Repair)
- `process_bootstrap_step1` (System Bootstrapping)
- 위 모든 주요 로직에서 언어 설정을 인지하고 프롬프트 최상단에 규약을 배치하도록 수정.

## ✅ 결과 (Results)
- `ko_KR` 환경에서 AI 요원이 자신의 생각을 영어로 뱉던 현상 해결.
- 시스템 전반의 메시지 일관성 확보.
