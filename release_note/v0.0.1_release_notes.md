# v0.0.1 Release Notes - Control & Isolation Hardening

## 🚀 Key Updates
1. **Struct Synchronization for Sandbox Registry**
   - Synchronized the `Daemon` struct with the memory-resident `SandboxRegistry` and `Notify` events.
   - Restored state routing integrity in `server.rs` by aligning `DaemonState::Committing` with explicit metadata fields (`task_id`, `started_at`).
   
2. **Deterministic Task & Thread Pipeline Realignment**
   - Corrected structural drift in `axon_core::Thread` and `axon_core::Task` initializers by populating newly added auditing fields (rejections, interventions, and feedback).
   - Migrated all task types from legacy `TaskKind` definitions to structured, multi-language `LanguageTaskKind` enums (specifically C-variant tasks like `HeaderDecl`, `SourceImpl`, and `Integrator`).
   
3. **Dependency-Aware Coordinator Batching**
   - Updated `Coordinator::build_batch` calls to resolve batch readiness by checking completed tasks in real-time from storage, mitigating circular dependency locks.

4. **Component Taxonomy Enforcement**
   - Integrated semantic ownership metadata (`generator_patchable`) and dependency imports inside the IR compiler using the proper `axon_ir` library.

## 🛠️ Technical File Changes
- **`crates/axon-daemon/src/lib.rs`**: Core changes to structures, initializers, and scheduler batch triggers.
- **`crates/axon-daemon/src/server.rs`**: Fixed API alignments for state transitions.
