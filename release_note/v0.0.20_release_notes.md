# 🚀 AXON Release Notes - v0.0.20

## [v0.0.20] - 2026-04-29

### ⚖️ Spec Drift Zero (IR-Driven Architecture)
- **SSOT IR Engine**: Replaced manual prompt and validator coding with a deterministic **Constraint IR** system. This ensures that what the Agent is asked to do (Prompt) is exactly what it is judged on (Validator).
- **Intelligent Conditional Rules**: Implemented context-aware validation. For example, the word `node` is now strictly forbidden in general text but fully allowed within `mermaid` dependency graphs.

### 🧠 Hierarchical Intelligence (The Three-Tier Memory)
- **Hot Rule Cache (Short-term)**: Instant failure feedback in memory (<1ms). Uses LRU and weighting with automatic decay to keep hints fresh and relevant.
- **Promotion Engine (Mid-term)**: A rigorous "Trial" system that validates hints through Scoring and Shadow Testing (Trace Replay) before promoting them to Local IR.
- **Global IR Registry (Long-term)**: Cross-project synchronization using Semantic Versioning (SemVer). Implements a `Global -> Profile -> Project Override` hierarchy for organizational law.

### ⚡ 1050TI Hardware Hardening
- **Stateless/Stateful Paradigm**: Achieved "Learning" without GPU-heavy fine-tuning or LoRA. The system learns through IR evolution while the LLM remains a lightweight generator.
- **Runtime Optimization**: Forced `num_ctx: 4096`, `stop: ["```"]`, and `temperature: 0.0` for maximum stability and VRAM efficiency.
- **Dynamic Failure Injection**: Recent errors are instantly fed back into the prompt to prevent repetitive mistakes.

### 🌿 Selective Rule Propagation
- **Upstream-Only Flow**: Rules now intelligently propagate from dependencies to their parents, ensuring consistent standards across the code hierarchy.
- **Shadow-First Policy**: All propagated rules are initially deployed in 'Shadow' mode to verify their impact in new contexts before final promotion.
- **Contagion Prevention**: Strict depth limits and type isolation prevent over-regulation and maintain development agility.

### 🔍 Identity & Intelligence
- **Full Model Synchronization**: Agents now correctly identify their assigned models (e.g., Qwen, Gemini) from the global configuration.
- **Dynamic Prompt Adaptation**: The system automatically switches to a high-efficiency, one-shot prompt template for smaller models (1.8B/2B) to ensure success in low-compute environments.

### 🛠️ Maintenance
- Unified recent development efforts into the v0.0.20 milestone.
- Removed legacy hard-coded strings and improved diagnostic logging for model detection.
