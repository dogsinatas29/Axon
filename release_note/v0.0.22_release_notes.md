# 🚀 AXON Release Notes - v0.0.22 (Hardened Reliability)

## [v0.0.22] - 2026-04-30

### 🇰🇷 한국어 (Korean)

**1. 엄격한 JSON 출력 규약 도입 (Strict JSON Output Contract)**
- 주니어 요원의 모든 코드 출력을 순수 JSON 배열 형식으로 강제했습니다. 이로써 마크다운 파싱 오류나 불필요한 설명 텍스트로 인해 파이프라인이 멈추는 현상을 원천 차단했습니다.

**2. Stage 3.5: 물리적 파일 Stub 생성 (Stub Generation)**
- 실제 구현 태스크가 시작되기 전, 설계도에 정의된 모든 파일의 뼈대(Stub)를 물리적으로 먼저 생성합니다. 이를 통해 병렬 작업 중 발생하던 의존성 임포트 에러(`ModuleNotFoundError`)를 해결했습니다.

**3. 스마트 매핑 검증기 (Smart Mapping Validator)**
- 엔트리 포인트 검증 로직을 유연하게 개선하여 `main()` 함수 외에도 설계도에 정의된 다양한 실행 함수를 수용합니다. 또한, 시뮬레이터가 전체 프로젝트 상태를 검증기에 전달하도록 하여 아키텍처 드리프트(Drift) 오탐지를 제거했습니다.

**4. UI 동기화 및 가시성 개선 (UI & Observability)**
- 모든 태스크가 Studio UI의 작업 게시판(Work Board)에 즉시 나타나도록 동기화 로직을 강화했습니다. 에이전트의 "생각 중/완료" 상태를 실시간 신호(Signals)로 스트리밍하여 투명성을 높였습니다.

**5. 유연한 의존성 하네스 (Flexible Dependency Harness)**
- 실행 환경의 의존성(pandas 등) 체크를 완화하여 라이브러리 미설치 시에도 전체 파이프라인이 중단되지 않고 유연하게 대처할 수 있도록 개선했습니다.

### 🇺🇸 English

**1. Strict JSON Output Contract Enforcement**
- Mandatory pure JSON array format for all Junior agent code emissions. This eliminates pipeline stalls caused by markdown parsing errors or unnecessary explanatory text.

**2. Stage 3.5: Physical File Stub Generation**
- Automatically materializes physical stubs for all files defined in the architecture before implementation begins. This resolves cross-file dependency errors (`ModuleNotFoundError`) during parallel bootstrapping.

**3. Smart Mapping Validator Enhancements**
- Upgraded entry point validation to accept various execution functions defined in the architecture beyond just `main()`. Improved the simulator to provide a full-state view to the validator, eliminating false-positive "Architecture Drift" detections.

**4. UI Synchronization & Observability Upgrades**
- Hardened synchronization logic to ensure all tasks appear instantly on the Studio UI Work Board. Implemented real-time signal streaming for "Thinking/Completed" agent states, providing maximum transparency.

**5. Flexible Execution Harness**
- Softened runtime dependency checks (e.g., pandas) to prevent global pipeline failures due to missing local libraries, allowing for better environment adaptability while maintaining core logic safety.
