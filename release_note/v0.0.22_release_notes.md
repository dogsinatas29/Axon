# v0.0.22 Release Notes - Deterministic Factory Hardening

## 🚀 Overview
본 버전은 AXON 공정의 결정론적 신뢰도를 임계점까지 끌어올리기 위한 **'무결성 강화'** 릴리즈입니다. 특히 소형 모델 사용 시 발생하는 '조용한 실패(Silent Failure)'를 원천 차단하고, 실제 물리 파일의 생성을 보장하는 데 초점을 맞추었습니다.

## 🛠️ 주요 변경 사항

### 1. Toolchain Hardening (Guardian Upgrade)
- **`axon_patch_simulator.py` (V-FS)**: 
    - 요원(Junior)의 제안서가 JSON 규격을 벗어날 경우(코드 직출력 등), 기존에는 에러 메시지만 반환하고 종료했으나 이제는 즉시 **`exit(1)`**을 수행하여 빌드 파이프라인을 중단시킵니다.
    - 이를 통해 잘못된 상태가 SSOT(Single Source of Truth)로 승격되는 것을 방지합니다.
- **`axon_ir_validator.py` (Semantic Guardian)**:
    - 시뮬레이터에서 발생한 에러 키를 명시적으로 감지하도록 로직을 보강하였습니다.
    - 이제 "Validation Success" 메시지는 오직 실제 코드가 무결하게 생성되었을 때만 출력됩니다.

### 2. Orchestration & Logistics
- **Legacy Parser Removal**: JSON 기반 워크플로우에서 불필요하게 호출되던 레거시 파서(`sync_post_to_sandbox`)를 제거하여 `Parser Tier 1/2 failed` 오탐지 로그를 완전히 박멸하였습니다.
- **SSOT Promotion Integrity**: 모든 파일 쓰기 권한을 `axon_registry.py`로 일원화하여 에이전트의 메모리와 실제 디스크 사이의 '스펙 드리프트(Spec Drift)'를 해결하였습니다.

### 3. Verification & QA
- **TEST3/GEMINI 프로젝트 정밀 검증**:
    - 생성된 소스 파일이 실제 로직이 아닌 Stub(마커) 상태임을 포착하였습니다.
    - 원인 분석 결과, 시뮬레이터의 침묵하는 에러 처리가 주범임을 밝혀내고 수정 완료하였습니다.

## ⚠️ Known Issues & Roadmap
- **Stub Re-run Required**: 현재 `TEST3/GEMINI`의 파일들은 Stub 상태입니다. 툴체인이 강화되었으므로, 데몬 재가동을 통해 실제 로직으로의 치환이 필요합니다.
- **Truncation Issue**: LLM 응답이 중간에 끊기는 문제는 다음 버전(v0.0.23)에서 본격적으로 다룰 예정입니다.
