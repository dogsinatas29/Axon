# AXON Release Notes - v0.0.23 (Phase 07)

## [KO] 요약: 물리적 무결성 및 시니어 거버넌스 강화
이번 버전은 AXON 공장의 '논리'와 '물리'를 완전히 연결하는 데 집중했습니다. 샌드박스에서만 돌아가던 코드가 이제 실제 프로젝트 디렉토리에 안전하게 배포(Materialize)되고, 실물 환경에서의 최종 검증을 거쳐 시니어의 승인을 받습니다.

### 주요 변경 사항
1. **물리 검증 파이프라인 (M4)**: 
   - 논리 승인 → 물리 커밋 → 물리 검증 → 시니어 최종 승인 순서로 파이프라인을 분리했습니다.
   - **시니어 개입 시점**: 이제 시니어는 기계가 검증한 '실물 코드'를 보고 최종 락인(Lock-in) 여부를 결정합니다. 물리 하네스 실패 시 시니어에게 즉시 인터럽트가 발생합니다.
2. **자동 롤백 (Auto-Rollback)**: 실물 하네스 검증 실패 시, 작업 디렉토리를 오염시키지 않도록 즉시 이전 상태로 파일을 원복합니다.
3. **Stub 박멸**: `# AXON STUB` 문구가 포함된 코드는 시뮬레이션 단계에서 즉시 거절됩니다.
4. **하네스 고도화 (v0.1)**: 
   - 파일 존재(F1), 무결성(F2), 진입점(F3), 사이드 이펙트(F9) 등을 전수 조사하는 정밀 검증 기능을 탑재했습니다.
5. **문서 최적화**: 
   - README 데모 영상의 "2.0배속" 문구를 제거하고 "Full pipeline demo"로 통일하여 시청 부담을 줄였습니다.
6. **라이선스 및 상호 링크**:
   - 라이선스를 MIT에서 GNU GPL v3로 전환하고 모든 문서를 업데이트했습니다.
   - README.md와 README.ko.md 간의 상호 링크를 추가하여 접근성을 높였습니다.
7. **개발 환경 명시**: 
   - AXON의 로컬 구동 성능을 검증하기 위해 사용된 하드웨어 사양(i7-4790, GTX 1050 Ti)과 로컬 LLM 환경(Ollama, qwen2.5)을 README에 명시했습니다.
8. **문서 구조 개편 및 기능 추가**:
   - 에러 진단 및 시니어 리뷰 프로토콜을 에이전트 정의 하위로 이동하여 문맥적 연결성을 강화했습니다.
   - **노가리 채널(Lounge)**, **페르소나 시스템**, **인사 게시판(Greetings Board)** 섹션을 신설하여 시스템의 고유 기능을 명세화했습니다.

## [EN] Summary: Physical Integrity & Senior Governance Hardening
This version focuses on bridging the gap between AXON's 'Logic' and 'Physical' layers. Code that previously only ran in a sandbox is now safely materialized into the real project directory, passing final physical environment validation before reaching the Senior's gate.

### Key Changes
1. **Physical Verification Pipeline (M4)**:
   - Split the pipeline: Logical Approval → Physical Commit → Physical Validation → Senior Final Gate.
   - **Senior Intervention Point**: Seniors now review the 'Real Code' verified by the machine to decide on final Lock-in. An interrupt is sent to the Senior immediately if the physical harness fails.
2. **Auto-Rollback**: If the physical harness fails, files are immediately reverted to their previous state to prevent workspace contamination.
3. **Stub Eradication**: Any code containing `# AXON STUB` is rejected immediately during the simulation stage.
4. **Harness Hardening (v0.1)**: 
   - Integrated precision checks for file existence (F1), integrity (F2), entry points (F3), and side-effects (F9).
5. **Documentation Optimization**:
   - Removed "2.0x speed" mentions from README demo videos and unified as "Full pipeline demo" to reduce viewer friction.
6. **License & Cross-links**:
   - Pivoted license from MIT to GNU GPL v3 and updated all relevant badges.
   - Added explicit cross-links between README.md and README.ko.md for better navigation.
7. **Environment Specification**:
   - Documented the exact hardware (i7-4790, GTX 1050 Ti) and local LLM setup (Ollama, qwen2.5) used for verification to provide a baseline for local-first execution.
8. **Restructuring & Feature Documentation**:
   - Reorganized Diagnostic and Review protocols under Agent definitions for better logical flow.
   - Formalized sections for **Lounge System (Nogari)**, **Persona-based Agents**, and the **Greetings Board**.
