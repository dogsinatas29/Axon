# AXON v0.0.24 Release Notes (Hardened Factory Pipeline)

## [KR] 주요 변경 사항

### 🛠️ 파이프라인 무결성 강화 (Hardening)
- **SSOT 드리프트 방지**: Python과 Rust 간의 줄바꿈 처리 차이로 인해 발생하던 허위 `Target Mismatch Violation` 문제를 Newline-agnostic 비교 로직으로 해결했습니다.
- **물리적 검증 임계치 완화**: 60B 미만의 짧은 핸들러 코드(Output, Persistence 등)가 기각되지 않도록 `WIPE_SHIELD` 및 하네스 제한을 120B -> 60B로 조정했습니다.
- **감사 추적(Audit) 강화**: 실패한 모든 제안서(Proposal)를 DB의 `posts` 테이블에 강제 저장하여 사후 분석이 가능하도록 개선했습니다.

### 🌐 연결성 및 가시성 개선
- **Connectivity Guard**: 초기 실행 시 로컬 LLM 엔드포인트(Ollama 등)의 연결 상태를 실시간 검증하는 루프를 도입했습니다.
- **진단 로그 상세화**: Mapping Validator의 파이썬 에러(`stderr`)를 데몬 로그에 통합하여 아규먼트 누락 등 내부 오류 가시성을 확보했습니다.

### 🤖 에이전트 행동 교정
- **Strict Isolation**: 시스템 프롬프트에 타겟 파일 외 수정 금지 규칙을 강화하고, 위반 시 즉시 반려(Instant Rejection) 페널티를 명시했습니다.

---

## [EN] Key Changes

### 🛠️ Pipeline Integrity Hardening
- **SSOT Drift Prevention**: Resolved false-positive `Target Mismatch Violations` caused by Python/Rust newline handling discrepancies using newline-agnostic comparison.
- **Validation Threshold Relaxation**: Adjusted `WIPE_SHIELD` and harness limits from 120B to 60B to accommodate valid short code stubs (e.g., Output, Persistence).
- **Audit Preservation**: Enhanced auditing by forcing all failed proposals into the `posts` table for post-mortem analysis.

### 🌐 Connectivity & Visibility
- **Connectivity Guard**: Implemented a real-time verification loop for local LLM endpoint (Ollama) accessibility during initialization.
- **Diagnostic Transparency**: Integrated Mapping Validator `stderr` into daemon logs to capture internal Python errors and missing arguments.

### 🤖 Agent Behavioral Correction
- **Strict Isolation**: Hardened system prompts to strictly forbid modifications outside the target file, with explicit 'Instant Rejection' penalties for violations.
