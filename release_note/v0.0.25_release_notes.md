# 🚀 AXON v0.0.25 Release Notes - "Intelligent Batch Pipeline"

## 🏗️ Overview
이번 릴리즈는 AXON 공장의 처리 효율과 정합성을 동시에 극대화하는 **지능형 배치(Batch) 시스템**을 도입했습니다. 이제 개별 태스크 단위의 파편화된 처리를 넘어, 의존성 클러스터 단위로 변경사항을 통합하고 원자적으로 검증합니다.

## 🌟 Key Changes
### 1. 배치 기반 스케줄링 (Batch-Aware Scheduling)
- **Dependency Clustering**: `DepGraph`를 분석하여 의존성이 얽힌 파일들을 하나의 `Batch`로 묶어 처리합니다.
- **Priority-Seed Builder**: 가장 높은 우선순위(Fan-in 기반)를 가진 태스크를 시드로 삼아 관련 태스크들을 배출구(Queue)에서 통합 추출합니다.

### 2. 지능형 선택적 검증 (Selective Validation)
- **NodeRole 도입**: 컴포넌트를 `Entry`, `Boundary`, `Pure`로 분류하여 런타임 검증 대상을 지능적으로 선별합니다.
- **Incremental Compile Gate**: `cargo check`와 파이썬 구문 검사를 점진적으로 수행하며, 10회 주기마다 Full Check를 통해 정합성을 강제 보정합니다.

### 3. 원자적 파이널 게이트 (Atomic Final Gate)
- **Global Serialization**: 전역 락을 통해 배치의 커밋 단위를 직렬화하여 동시성 문제를 원천 차단합니다.
- **Rollback & Fan-out**: 배치 내 단 하나의 태스크라도 검증에 실패할 경우, 전체 배치를 롤백하고 영향 범위 내의 모든 파일을 연쇄 리워크 대상으로 등록합니다.

## 🛠️ Technical Improvements
- `intelligence::coordinator::Coordinator`: 배치 빌더 및 우선순위 스코어링 로직 강화.
- `execution_validator`: 증분/전체 검증 모드 및 선택적 런타임 실행 지원.
- `Daemon::handle_assignment`: 배치 단위의 원자적 처리 및 롤백 매커니즘 구현.

## ⚠️ Notes
- 멀티 워커 환경에서의 샌드박스 정합성을 위해 일시적으로 워커 수를 1로 제한하여 안정성을 확보했습니다. (향후 Isolation 계층 강화 후 확장 예정)
- `architecture.md`의 의존성 클로저가 클수록 배치 규모가 커지며, 이는 대규모 리팩토링 시의 정합성 유지에 유리합니다.
