# 🚀 AXON v0.0.29 Release Notes - Observability & Signal Hardening

---

## 🏛️ Summary
이번 업데이트의 핵심은 **"공장 내부의 투명성 확보"**와 **"파이프라인의 결정론적 안정화"**입니다. 이제 터미널을 보지 않고도 브라우저(Studio UI)만으로 데몬의 모든 로그와 치명적인 에러를 실시간으로 모니터링할 수 있으며, UI 로컬라이제이션 완성으로 작업 편의성을 극대화했습니다.

## 🛠️ 주요 변경 사항

### 1. UI 로컬라이제이션(다국어) 완성 및 동기화
- **전면 적용:** `ThreadDetail`, `ThreadCard`, `Office`, `BossBoard`, `Lounge` 등 모든 주요 UI 컴포넌트에 로컬라이제이션을 적용하였습니다.
- **실시간 동기화:** `App.tsx`를 통해 `t` 객체를 전파하는 구조로 개선하여 언어 전환 시 UI 전체가 즉시 업데이트됩니다.
- **다국어 지원:** 한국어, 영어, 일본어 번역 키를 추가하여 글로벌 환경에서의 가독성을 확보했습니다.

### 2. 이중 승인 문제 해결 (부분 재진입 로직)
- **Anti-Double Approval:** 배치 작업 중 일부가 실패할 경우, 성공한 작업은 유지하고 **실패/반려된 태스크만 선택적으로 재진입(Re-queue)**시키도록 로직을 고도화하였습니다.
- **효율성 증대:** 불필요한 Senior Review 재수행을 방지하여 토큰 소모를 줄이고 파이프라인 진행 속도를 높였습니다.

### 3. 결정론적 헤더 보호 강화 (Header Hardening)
- **Deterministic Header Gate [HARDENED]**: 헤더(.h) 파일 내 함수 본문 포함 시 즉시 반려하는 물리적 가드를 더욱 정교하게 강화하였습니다.
- **시스템 프롬프트 보강**: 주니어 요원에게 헤더 가드 필수 작성 및 선언 전용 규칙을 '절대 규칙'으로 주입하여 C 빌드 안정성을 확보했습니다.

### 4. 실시간 로그 브릿지 (Tracing to UI Bridge)
- **`EventBusLayer` 도입:** Rust의 `tracing` 시스템과 AXON의 `EventBus`를 직접 연결하여 데몬 로그를 브라우저에서 실시간 관제할 수 있습니다.
- **시각적 계층화:** `EventLevel` 프로토콜을 통해 크리티컬 에러(❌) 발생 시 UI 쉐이크 애니메이션 등 시각적 경보를 제공합니다.

## 🧪 검증 결과
- `cargo check`: 모든 모듈 정상 컴파일 완료.
- `Studio UI`: 다국어 전환 및 실시간 로그 수신/에러 알림 동작 확인.
- `C Build Pipeline`: `validators.h` 등 헤더 파일의 정합성 및 중복 정의 방지 확인.
