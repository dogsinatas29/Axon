# AXON v0.0.31.34 Release Notes - Task Decomposition Hardening

본 릴리즈는 `TEST4_GTK` 프로젝트 바인딩 시 핵심 소스 코드 파일들이 단일 태스크로 오염/축소되는 교착 오작동을 해결하고 시맨틱 팩토리 엔진의 수송 상태 정리를 완료한 내역을 담고 있습니다.

---

## 🚀 주요 변경 사항

### 1. [BUGFIX] `already_present` 중복 매칭 논리 엄격화
- 기존 `crates/axon-daemon/src/lib.rs` 내 `already_present` 검사에서 `contains` 부분 매치로 인해 missing core components가 강제 주입되지 못하고 스킵되던 버그를 완전 일치(`==`)로 변경하여 해결했습니다.
- 이를 통해 `immutable_constraints.json`에 정의된 핵심 소스 파일들이 LLM IR 빌드 시점에 누락되더라도 시딩 로직(`[IR_SEED]`)이 안정적으로 활성화되도록 보장합니다.

### 2. [OPERATION] 공유 runtime/state.db 리셋
- 각 샌드박스의 DB 격리 한계를 보완하기 위해 워크스페이스 루트의 `runtime/state.db`를 리셋하고 기존 좀비 데몬 프로세스를 정리함으로써, 복구 모듈의 과거 스냅샷 교착 유입을 원천 차단했습니다.

### 3. [ENGINE] 릴리즈 빌드 튜닝
- `target/release/axon-daemon` 릴리즈 버전을 컴파일하여 실제 런타임 상의 LSP 및 VRAM 오버헤드를 낮추고, 병렬 스케줄러 처리량을 극대화했습니다.
