# 🚀 Release Notes - v0.0.31.50

---

## 🛠️ v0.0.31.50 - Deterministic Normalizer Layer (Truth Authority) (Phase 23)

### 1. 개요 및 설계 철학
- **배경**: 기존 LLM 컴파일러 프론트엔드가 자연어 명세서로부터 엄격한 `ImmutableConstraints` JSON 스키마를 직접 온전하게 추출하도록 강제하면서, 7B 소형 로컬 추론 모델이나 7B Instruct 모델 환경에서 스키마 불일치(Deserialize Error)나 키 소실, 또는 엉뚱한 필드 명명 오류로 전체 파이프라인이 붕괴되는 현상이 빈번히 발생했습니다.
- **철학**: **"좋은 시스템은 모델을 똑똑하게 만드는 게 아니라, 모델이 바보여도 못 벗어나게 만든다"**
  - LLM의 역할을 자유도가 높은 러프한 의미론적 정보 수집기(Lexer & Semantic Hint Engine)로 한정하여 제약을 극도로 축소시킵니다.
  - Rust가 최종 데이터 해석의 권한(Truth Authority)을 확보하고, 입력 명세 원본(`spec.md`)의 문자열 기계적 매칭과 디폴트 안전망 필터를 결합하여 빈틈없는 JSON 객체를 강제로 빌드 및 정규화(Deterministic Normalizer Layer)합니다.

---

### 🌟 4대 핵심 아키텍처 및 구현 내역

#### 1. Robust Fuzzy JSON Parsing & Fallback
- LLM의 텍스트 생성 중 일부 괄호가 누락되거나 JSON 포맷이 깨지더라도 파이프라인이 즉시 에러로 중단되지 않도록 fuzzy fallback 처리를 구축했습니다.
- 파싱 실패 시 경고를 남기고 빈 JSON 객체(`{}`)를 반환하여 Rust Normalizer가 주도적으로 구조를 완성할 수 있는 안전 발판을 확보했습니다.

#### 2. Platform & Language Deterministic Normalization
- 명세 원본 `spec.md`를 직접 Rust 단에서 스캔하여, `gtk` / `cairo` 키워드 존재 시 `platform = "linux"`, `subsystem = "gtk4"`, `runtime_model = "gtk4"`, `language = "c"`로, `win32` / `windows.h` 키워드 존재 시 `platform = "win32"`, `subsystem = "windowsgui"`, `runtime_model = "win32gui"`, `language = "cpp"`로 강제 교정 및 정규화합니다.

#### 3. Component Constraint Merging & Safe Guard
- LLM이 추출한 컴포넌트 목록에 대해 필수 필드(file_path, status, promotion_forbidden 등)를 누락 없이 디폴트 값으로 정규화합니다.
- Rust가 정규식으로 기계 추출한 Baseline 컴포넌트 목록(`deterministic_extract_components`)과 LLM이 임의로 뱉어낸 힌트 목록을 안전하게 병합(Safe Merge)하여, LLM 추출 누락으로 인한 파일 유실 버그를 원천적으로 극복했습니다.

#### 4. Mandatory Constitutional Patterns Force-Injection
- AXON 헌법상 강제되는 기본 금지 패턴(`sqlite3`, `libpq`, `mysqlclient`, `ncurses`, `WinMain`, `HWND` 등)과 코어 런타임 목록을 Rust 코드 단에서 `forbidden_patterns` 및 `runtime_core` 에 무조건 주입하도록 하드닝했습니다.
