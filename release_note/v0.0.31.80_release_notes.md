# AXON v0.0.31.80 Release Notes - Lossless Section Fallback Parser & Rationalized Constraints

---

## 🏛️ Sovereign Protocol & Compilation Hardening
**"시스템의 오류 발생 시에도 원본 사양의 정보 밀도는 100% 무손실로 보전되어야 한다."**

### 1. 주요 변경 사항 및 엔지니어링 의도

* **무손실 문단 기반 Section Parser (`fallback_harvest_spec` 개편)**:
  - 기존의 단순 라인 스캔 파서는 `must`, `forbidden` 등의 키워드가 포함된 단일 라인만 조각내어 가져갔기 때문에, 그에 종속된 목록(`* Windows GUI...`)이나 상세 코드 블록(CMake, C++) 등 본문 알맹이가 대거 소실되는 결함이 있었습니다.
  - 마크다운 헤더(`#`, `##`, `###`)를 기준으로 문서를 온전히 나누어 각 헤더와 그에 속하는 하위 바디 텍스트 전체를 하나의 `SemanticFragment`로 묶어 조립하는 구조적 구문 파서를 구현했습니다.
  - 이를 통해 LLM Extractor가 로컬 저사양 모델 환경에서 Reject 되더라도, Fallback 컴파일러가 원본 `spec.md`를 100% 무손실 상태로 분할/라우팅하여 `spec/` 아래 4대 Canonical Spec 문서로 완성합니다.

* **검증 제약 합리화 (`validate_semantic_fragments`)**:
  - LLM Harvester가 매번 Reject를 겪지 않도록, 지나치게 파편화되어 있던 8대 필수 키워드 대조 필터를 실질적 코어 키워드 4대(`GtkApplicationWindow`, `lua_State`, `cairo_t`, `CMakeLists.txt`)로 합리적으로 정돈했습니다.
