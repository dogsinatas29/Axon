# Release Notes - v0.0.31.90: Deterministic Project ID Resolution

## 🌟 개요
- **버전:** `v0.0.31.90` (Phase 24 - Step 4)
- **목표:** `spec.md` 등 범용적인 파일명으로 부트스트랩 기동 시 발생하는 `project_id` 및 경로 상실 오류를 잡고, 외부 링킹 라이브러리들이 local source code 컴포넌트로 오염(contamination)되던 문제를 원천 격파.

## 🛠️ 세부 변경 내용

### 1. `crates/axon-daemon/src/lib.rs` (Project ID 동적 분기 처리)
- `bootstrap_from_spec` 함수에서 `spec_path`의 file_stem을 바로 `project_id`로 지정하던 로직을 보완했습니다.
- 만약 file_stem이 `"spec"` 또는 `"specification"`과 같이 범용 어휘일 경우, 해당 사양서 파일의 부모 디렉토리 이름(예: `TEST4_GTK`)을 `project_id`로 자동 치환합니다.
- 이에 따라 `self.sandbox_root`가 오염되었던 임시 경로 `spec/`이 아닌 `TEST4_GTK/`를 정확하게 가리키게 되었습니다.

### 2. 컴포넌트 시맨틱 오염(Semantic Contamination) 해소
- 올바른 프로젝트 디렉토리를 참조함에 따라, `TEST4_GTK/immutable_constraints.json`에 정의되어 있던 올바른 6대 C++17 코어 컴포넌트(`src/main.cpp`, `src/gtk_app.cpp` 등) 캐시를 성공적으로 불러왔습니다.
- 저사양 7B 모델이 `target_link_libraries`의 링킹 인자(`pthread`, `cairo`, `libc`)를 로컬 소스 파일(`src/pthread.cpp`, `src/cairo.cpp` 등)로 잘못 추출해 내던 시맨틱 오염이 완전히 격하 및 바이패스되었습니다.
