# v0.0.31.92 Release Notes - Axon Governance & Parser Hardening

본 릴리즈는 AXON 소프트웨어 공장 파이프라인의 핵심 안정성(Reliability)과 원활한 작업 실체화(Materialization) 흐름을 방해하던 두 가지 주요 병목을 완벽히 해결하는 마일스톤 패치입니다.

---

## 🏛️ 주요 개선 사항 및 해결된 문제

### 1. JSON 단일 객체 sequence 강제 포장 (Canonicalizer 2차 구제 규칙)
- **현상**: LLM(특히 qwen2.5)이 단일 태스크만을 생성할 때 JSON 배열(`[...]`)이 아닌 단일 JSON 객체(`{...}`)만으로 응답을 출력하는 포맷 변형이 있었습니다. 이로 인해 파서가 `invalid type: map, expected a sequence` 역직렬화 에러를 유발하며 파이프라인을 중단하고 이전 단계로 되돌려 보내는 무한 루프가 발생했습니다.
- **해결**: `Stage::HeaderGen` 및 `Stage::ImplGen` 각각에 대칭적인 Canonicalizer 2차 복구 규칙을 도입했습니다. LLM의 출력 블록이 단일 객체이고 `title` 또는 `component_id`와 같은 태스크 필드가 식별될 경우, 이를 `serde_json::Value::Array(vec![raw_val])` 형태의 단일 원소 시퀀스 배열로 즉시 Wrapping하여 파싱 안전성을 견고하게 보호합니다.

### 2. 워커 권한 세마포어 상태 파싱 개선 (`check_generation_permission`)
- **현상**: 보스 승인이 REST API(`SEAL`)를 통해 정상 수락되어 `.axon_approval_pending` 파일 내부의 `approved` 속성이 `true`로 승인 완료 갱신되었음에도 불구하고, 워커 스레드는 단지 해당 파일의 물리적 존재 여부(`exists()`)만을 검사하여 `GENERATION_LOCKED` 권한 위반 에러를 무작정 유발했습니다. 이로 인해 파이프라인 스케줄러가 코드를 빌드하고 실체화하는 단계로 진입할 수 없었습니다.
- **해결**: 파일이 존재하더라도 내부 JSON 콘텐츠를 분석하여 `approved == true`인 경우 승인 상태로 인지하고 락을 안전히 해제(`🔓 [GENERATION_UNLOCKED]`)한 채 작업을 진행하도록 permission check 조건을 완벽하게 고쳤습니다.

### 3. 사장 게시판 내 Factory Gateway 중복 노출 결함 해결 (`get_semantic_risks`)
- **현상**: 보스 승인이 이미 완료(`approved: true`)되어 워커가 작업을 돌리고 있는 상황에서도, 사장 게시판(SEMANTIC GOVERNANCE CONSOLE) 상에 `Factory Gateway` 카드가 계속 남아있는 혼선이 발생했습니다.
- **해결**: `get_semantic_risks` API가 승인 여부를 검증하여 `approved == true`인 경우 대기 리스크 목록에서 카드를 정상적으로 지워주도록 필터링을 추가했습니다.
