# AXON Release Notes - v0.0.32

## 🚀 Key Improvements: Factory Visibility & Precision

### 1. Hardened Architectural Extraction (Fix Problem 1)
- **Problem**: The Architect agent was truncating specifications, often only generating 2 tasks (main.c/utils.h) regardless of the actual requirements.
- **Solution**: Re-engineered the `SKELETON_SYSTEM_PROMPT` in `AgentRuntime`.
  - Replaced hardcoded examples with a generic, mandatory schema.
  - Added a `CRITICAL` instruction enforcing the extraction of EVERY module, function, and header defined in the `spec.md`.
  - Result: All components (`database.c/h`, `calculator.c/h`, `validators.c/h`) are now correctly identified and enqueued.

### 2. Deep UI Persistence (Fix Problem 2)
- **Problem**: Task threads on the Studio UI Work Board appeared empty, lacking junior code proposals and senior reviews.
- **Solution**: Integrated the `axon-storage` Post persistence into the daemon's materialization pipeline.
  - **Instruction Posts**: Every thread starts with a clear Boss instruction.
  - **Proposal Posts**: Junior agent code proposals (including full source) are now saved as persistent Posts.
  - **Review Posts**: Senior agent approval/rejection feedback is now visible in the thread.
- **Result**: The "Battle Log" between AI agents is now fully observable in the right-side panel of the Studio UI.

### 3. Materialization & Reliability
- **Physical Commit Fix**: Restored the `std::fs::write` logic that was accidentally bypassed, ensuring code is physically committed to the `src/` directory.
- **Borrow Checker Repair**: Fixed a critical move error in `handle_assignment` involving the `metrics` object, ensuring system stability.
